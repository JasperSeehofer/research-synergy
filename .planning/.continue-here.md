<decisions_made>

- **Semantic Scholar over OpenAlex**: S2 returns `externalIds.ArXiv` on every reference directly. OpenAlex only returns opaque work IDs requiring N+1 lookups.

- **External governor at 2200ms (unkeyed) / 400ms (keyed)**: Each paper = 2 API calls. At S2's 1 rps limit, 2200ms/paper keeps total rate ≤ 1 rps. Internal `rate_limit_check` zeroed (per-task fresh source means it was a no-op anyway — matches InspireHep pattern).

- **Old-format arXiv IDs skip Link creation**: `Reference::get_arxiv_id()` splits URL by `/` and takes last segment → strips the cond-mat/ category prefix. For IDs containing `/`, only `arxiv_eprint` is set (correct fallback path). New-format IDs (no `/`) get both Link and arxiv_eprint.

- **Fresh DB at `./data-kuramoto`**: Existing `./data` has 383 LIGO papers (wrong domain). Left untouched.

<context>
The pre-2015 cond-mat citation crawler was fundamentally blocked because journal-only bibliography references have no arXiv links in HTML. The fix is a new `SemanticScholarSource` that queries S2's API which returns resolved arXiv IDs for ALL eras. The implementation is clean and fully additive — existing arxiv and inspirehep sources unchanged.

The key subtlety discovered during implementation: old-format arXiv IDs like `cond-mat/0010317` must NOT be wrapped in a `Link` object. The `Reference::get_arxiv_id()` function splits URL by `/` and takes the last segment, which returns only `0010317` (losing `cond-mat/`). The `arxiv_eprint` field fallback returns the full ID correctly. This is a pre-existing bug in the URL parsing that only surfaces for old-format IDs — new-format IDs (`2301.12345`) work fine with Links.

The Kuramoto notebook only needs pair papers to exist as nodes in different communities — it does NOT need edges between them. So the test is: do the 10 Feynman pair papers appear in the export as non-Other community nodes?
</context>
