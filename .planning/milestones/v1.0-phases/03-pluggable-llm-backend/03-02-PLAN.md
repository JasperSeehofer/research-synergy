<task type="auto" tdd="true">
  <name>Task 1: ClaudeProvider with wiremock integration tests</name>
  <files>src/llm/claude.rs, src/llm/mod.rs</files>
  <behavior>
    - ClaudeProvider::new(client, api_key, model) constructs provider with given params
    - ClaudeProvider::new reads ANTHROPIC_API_KEY from env if not passed directly (constructor helper)
    - ClaudeProvider sends POST to https://api.anthropic.com/v1/messages with headers: x-api-key, anthropic-version: 2023-06-01, content-type: application/json
    - Request body contains model, max_tokens: 1024, system (SYSTEM_PROMPT), messages with user content = abstract_text
    - Successful response: parses content[0].text as JSON into LlmAnnotation fields, sets provider="claude" and model_name
    - Parse failure: retries once with RETRY_NUDGE appended to system prompt, then returns ResynError::LlmApi on second failure
    - HTTP 429: returns ResynError::LlmApi with rate limit message
    - HTTP 4xx/5xx: returns ResynError::LlmApi with status code
    - Rate limiting: sleeps 1 second between calls using tokio::time::sleep + Instant tracking
  </behavior>
  <action>
    1. Create `src/llm/claude.rs` with `ClaudeProvider` struct:
       - Fields: `client: reqwest::Client`, `api_key: String`, `model: String`, `rate_limit: Duration` (default 1s), `last_called: Option<Instant>`
       - Constructor: `pub fn new(client: reqwest::Client) -> Result<Self, ResynError>` — reads `ANTHROPIC_API_KEY` from env, returns `ResynError::LlmApi("ANTHROPIC_API_KEY not set")` if missing. Default model: `"claude-haiku-4-5-20250415"`.
       - `pub fn with_model(mut self, model: String) -> Self` — builder pattern for model override.
       - `pub fn with_rate_limit(mut self, duration: Duration) -> Self` — builder pattern matching InspireHepClient.
       - Private `async fn rate_limit_check(&mut self)` — if `last_called` is Some and elapsed < rate_limit, sleep the difference. Update `last_called` after.
       - Private `async fn call_api(&self, system: &str, user_content: &str) -> Result<String, ResynError>` — POST to `https://api.anthropic.com/v1/messages` with all three required headers. Parse response as `ClaudeResponse { content: Vec<ClaudeResponseContent> }` where `ClaudeResponseContent { type: String, text: String }`. Return the text from first "text" content block.
       - `annotate_paper` impl: call rate_limit_check, call_api with SYSTEM_PROMPT and abstract_text, parse response as serde_json::from_str for the annotation fields (paper_type, methods, findings, open_problems — NOT the full LlmAnnotation since arxiv_id/provider/model_name/annotated_at are set by the caller). On parse failure, retry with `format!("{SYSTEM_PROMPT}\n\n{RETRY_NUDGE}")` and log raw response at debug level. On second parse failure, log warning and return error. Construct final `LlmAnnotation` with arxiv_id, provider="claude", model_name from self.model, annotated_at from chrono.

    2. Define helper deserialization structs:
       - `ClaudeResponseContent { #[serde(rename = "type")] content_type: String, text: String }`
       - `ClaudeResponse { content: Vec<ClaudeResponseContent> }`
       - `LlmAnnotationRaw { paper_type: String, methods: Vec<Method>, findings: Vec<Finding>, open_problems: Vec<String> }` — the partial shape the LLM returns (without provider metadata).

    3. Add `pub mod claude;` to `src/llm/mod.rs`.

    4. Write wiremock integration tests (in `#[cfg(test)] mod tests` inside claude.rs):
       - `test_claude_sends_correct_headers`: mock POST /v1/messages, verify x-api-key and anthropic-version headers present
       - `test_claude_parses_valid_response`: mock returns valid ClaudeResponse with JSON annotation text, verify LlmAnnotation fields
       - `test_claude_handles_429`: mock returns 429, verify ResynError::LlmApi returned
       - `test_claude_retries_on_parse_failure`: first mock returns non-JSON text, second mock returns valid JSON — verify success on retry
       All tests use `ClaudeProvider` constructed with wiremock server URL replacing api.anthropic.com (add a `with_base_url` builder for testing).
  </action>
  <verify>
    <automated>cargo test claude -- --nocapture</automated>
  </verify>
  <done>ClaudeProvider sends correct HTTP requests to Anthropic API, parses responses, handles errors, retries on parse failure, and rate-limits. All verified via wiremock tests without actual API calls.</done>
</task>
