    3. Create `src/gap_analysis/similarity.rs`:
       - `pub fn cosine_similarity(a: &HashMap<String, f32>, b: &HashMap<String, f32>) -> f32` — iterate shorter vector's keys, look up in longer, compute dot product / (mag_a * mag_b). Return 0.0 if either magnitude is zero.
       - `pub fn shared_high_weight_terms(a: &HashMap<String, f32>, b: &HashMap<String, f32>, min_weight: f32) -> Vec<String>` — terms present in both vectors with weight >= min_weight in both. Sort alphabetically for deterministic output.

    4. Create `src/gap_analysis/contradiction.rs`:
       - `pub async fn find_contradictions(analyses: &[PaperAnalysis], annotations: &[LlmAnnotation], provider: &mut dyn LlmProvider) -> Vec<GapFinding>`
       - Two-stage pipeline per CONTEXT.md:
         a. Stage 1 — TF-IDF similarity: For each unique pair of papers, compute cosine similarity of their tfidf_vectors. Keep pairs with similarity >= 0.3 (Claude's discretion threshold).
         b. Stage 2 — Finding strength divergence: For each candidate pair, look up their LlmAnnotation findings. If any finding strengths diverge (one paper has "strong"/"established" while the other has "weak"/"preliminary" on overlapping topic terms), mark as contradiction candidate.
         c. Stage 3 — LLM verification: Call `provider.verify_gap(CONTRADICTION_SYSTEM_PROMPT, context)` where context includes both papers' summaries and their divergent findings. If response starts with "NO", skip. Otherwise use response as justification.
       - Confidence score: Use the cosine similarity value of the pair (higher overlap = higher confidence they discuss the same topic).
       - On LLM failure (Err from verify_gap): log warning and skip the pair — "never block on missing data" per Phase 1 philosophy.
       - Shared terms: Extract from `shared_high_weight_terms(a, b, 0.1)` for display.
       - Inline `#[cfg(test)]` module — unit tests use mock data (no LLM needed for stage 1+2 tests; LLM stage tested via noop in integration).
  </action>

  </verify>

<task type="auto" tdd="true">
  <name>Task 2: ABC-bridge discoverer with graph distance</name>
  <files>src/gap_analysis/abc_bridge.rs</files>
  <behavior>
    - Test: find_abc_bridges discovers bridge where A and C share high-weight terms but don't directly cite each other (graph distance >= 2)
    - Test: find_abc_bridges excludes pairs that directly cite each other (edge A->C or C->A exists)
    - Test: find_abc_bridges returns empty when no non-citing pairs share high-weight terms
    - Test: graph_distance returns None for disconnected nodes
    - Test: graph_distance returns correct hop count for connected nodes
  </behavior>
  <action>
    1. Create `src/gap_analysis/abc_bridge.rs`:
       - Internal helper `fn graph_distance(graph: &StableGraph<Paper, f32, Directed>, from: NodeIndex, to: NodeIndex) -> Option<u32>`:
         Use `petgraph::algo::dijkstra` with unit edge weight. Per RESEARCH.md Pitfall 1: citation edges are directed but "connected by some path" means undirected reachability. Run dijkstra from both directions (from->to and to->from), return the minimum distance found. If neither direction finds a path, return None.
       - Internal helper `fn has_direct_edge(graph: &StableGraph<Paper, f32, Directed>, a: NodeIndex, b: NodeIndex) -> bool`:
         Check `graph.find_edge(a, b).is_some() || graph.find_edge(b, a).is_some()`.
       - `pub async fn find_abc_bridges(analyses: &[PaperAnalysis], annotations: &[LlmAnnotation], graph: &StableGraph<Paper, f32, Directed>, provider: &mut dyn LlmProvider, full_corpus: bool) -> Vec<GapFinding>`:
         a. Build a lookup map from arxiv_id -> NodeIndex by iterating graph nodes.
         b. For each unique pair of papers (A, C) where A and C do NOT have a direct edge:
            - If `full_corpus` is false, skip pairs where neither paper is in the graph (default scope = within citation graph only).
            - Compute `shared_high_weight_terms(a.tfidf_vector, c.tfidf_vector, 0.1)`. Need >= 3 shared high-weight terms (Claude's discretion) to qualify as having a B-intermediary connection.
            - Minimum graph distance for "non-obvious": >= 2 in undirected graph (per RESEARCH.md recommendation). If graph distance is 1 (direct citation), skip. If papers are in the graph but disconnected (distance None), they still qualify if full_corpus mode.
         c. LLM verification: Call `provider.verify_gap(ABC_BRIDGE_SYSTEM_PROMPT, context)` with context including both papers' summaries, their shared terms, and the intermediary B-connection description. If "NO", skip. Otherwise use as justification.
         d. Confidence score: `min(shared_terms.len() as f32 / 10.0, 1.0)` — normalized shared term count (per RESEARCH.md recommendation).
         e. On LLM failure: warn and skip (consistent with contradiction detector).
       - Inline `#[cfg(test)]` module — build small test graphs with `StableGraph`, test distance computation and bridge detection logic without LLM.
  </action>

  </verify>
