<task type="auto">
  <name>Task 2: CLI flags + run_gap_analysis pipeline wiring</name>
  <files>src/main.rs</files>
  <action>
    1. Add CLI flags to `Cli` struct:
       - `--full-corpus` (`#[arg(long, default_value_t = false)]`): Expand ABC-bridge scope to all papers in SurrealDB
       - `--verbose` (`#[arg(long, default_value_t = false)]`): Show full justifications in gap output table
       - Both are `bool` flags with default `false`.

    2. Update `run_analysis()` signature to accept `full_corpus: bool` and `verbose: bool` parameters. Thread these from `main()` through to `run_analysis()` calls (both the normal flow and db_only flow).

    3. Create `async fn run_gap_analysis(db: &Db, provider: &mut dyn LlmProvider, full_corpus: bool, verbose: bool)`:
       - Load all analyses: `AnalysisRepository::new(db).get_all_analyses().await`
       - Load all annotations: `LlmAnnotationRepository::new(db).get_all_annotations().await`
       - If either is empty, log info "No analyses/annotations found, skipping gap analysis" and return.
       - Corpus fingerprint cache guard (per RESEARCH.md pattern):
         * Compute fingerprint from sorted annotation arxiv_ids using `nlp::tfidf::corpus_fingerprint()`
         * Check `analysis_repo.get_metadata("gap_analysis").await` — if fingerprint matches, log "Gap corpus unchanged, skipping gap analysis" and fall through to display existing findings (do NOT return early — still display cached results from DB).
       - Build citation graph from papers: `PaperRepository::new(db).get_all_papers().await` then `create_graph_from_papers(&papers)`.
       - Call `gap_analysis::contradiction::find_contradictions(&analyses, &annotations, provider).await`
       - Call `gap_analysis::abc_bridge::find_abc_bridges(&analyses, &annotations, &graph, provider, full_corpus).await`
       - Persist all findings via `GapFindingRepository::new(db)`:
         * For each finding, call `gap_repo.insert_gap_finding(&finding).await` — log warning on error, don't block.
       - Update corpus fingerprint metadata with key `"gap_analysis"` via `analysis_repo.upsert_metadata()`.
       - Display: Load all findings from DB (including historical) via `gap_repo.get_all_gap_findings().await`.
       - Print `format_gap_table(&all_findings, verbose)` to stdout.
       - Print `format_gap_summary(&all_findings)` via `info!()`.

    4. Wire `run_gap_analysis` into `run_analysis()`:
       - Inside the existing `if let Some(provider_name) = llm_provider` block, AFTER `run_llm_analysis(db, provider.as_mut()).await;`, add: `run_gap_analysis(db, provider.as_mut(), full_corpus, verbose).await;`
       - Per RESEARCH.md anti-pattern: If `--llm-provider` is NOT specified, gap analysis is skipped entirely (it lives inside the provider block). This is correct — gap analysis requires LLM for verification.

    5. Handle `--full-corpus` without `--db`: If `full_corpus` is true and the paper list from DB is identical to the crawled set (no additional papers from prior crawls), log info "--full-corpus specified but no additional papers in DB beyond current crawl" — this is informational, not an error.
