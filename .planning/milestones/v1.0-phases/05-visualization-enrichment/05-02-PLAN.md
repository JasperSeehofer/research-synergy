**1. Node color and size application (in update loop):**

Add a method `fn apply_enrichment(&mut self)` to DemoApp that iterates all node weights:

```rust
fn apply_enrichment(&mut self) {
    let enriched = self.settings_analysis.enriched_view;
    self.g.g.node_weights_mut().for_each(|node| {
        if enriched {
            let arxiv_id = self.node_id_map.get(&node.id()).cloned().unwrap_or_default();
            if let Some(ann) = self.annotations.get(&arxiv_id) {
                node.set_color(enrichment::paper_type_to_color(&ann.paper_type));
                node.display_mut().radius = enrichment::finding_strength_radius(&ann.findings, enrichment::BASE_RADIUS);
            } else {
                node.set_color(enrichment::GRAY_UNANALYZED);
                node.display_mut().radius = enrichment::BASE_RADIUS;
            }
        } else {
            node.set_color(enrichment::DEFAULT_NODE_COLOR);
            node.display_mut().radius = enrichment::BASE_RADIUS;
        }
    });
}
```

Call `self.apply_enrichment()` in `App::update()` AFTER `self.sync()` and BEFORE the graph is rendered by DefaultGraphView.

**IMPORTANT**: The `apply_enrichment` call must happen before the `CentralPanel` rendering block. Restructure the `update()` method so that simulation sync + enrichment happen before the UI rendering. Currently the right panel renders first, then CentralPanel, then events/sync/simulation. The enrichment should be applied after sync but the node colors need to be set before the graph widget renders. The current order in update() is: SidePanel -> CentralPanel -> handle_events -> sync -> update_simulation -> update_fps. Since enrichment modifies node properties that the graph widget reads during CentralPanel rendering, either:
- Move sync + enrichment to happen BEFORE CentralPanel (restructure update order), OR
- Accept one-frame lag (enrichment applied after render, visible next frame) — this is acceptable for 60fps and simpler to implement.

The one-frame lag approach is recommended: call `self.apply_enrichment()` right after `self.sync()` at the end of update(). The visual effect is imperceptible at normal frame rates.

**4. Hover tooltip (MUST show paper title per locked decision):**

Implement `fn find_hovered_node(&self, ctx: &Context) -> Option<String>` on DemoApp:

Since egui_graphs 0.25.0 has NO hover events (RESEARCH.md Pitfall 1), use pointer position + distance check:

1. Get pointer position: `ctx.pointer_hover_pos()`
2. Iterate `self.g.g.node_weights()`, for each node get `node.location()` (graph-space position)
3. The coordinate transform between screen-space and graph-space depends on pan/zoom. Since `self.pan` and `self.zoom` from events may not perfectly inverse-transform (RESEARCH.md Pitfall 2, Open Question 3), try the direct approach first:
   - Convert pointer to graph space: `graph_pos = (screen_pos - Vec2::new(pan[0], pan[1])) / zoom`
   - Check distance: `graph_pos.distance(node.location()) < node.display().radius`
4. If the transform is inaccurate (tooltip fires for wrong nodes), fall back to using a generous hit radius and testing empirically. Log a warning if the transform seems off.
5. Return the arxiv_id of the nearest node within radius, or None.

In `App::update()`, AFTER the CentralPanel renders the graph, add tooltip rendering:

```rust
if self.settings_analysis.enriched_view {
    if let Some(hovered_id) = self.find_hovered_node(ctx) {
        // Look up the paper title from node_title_map (built in Plan 01 from Paper.title)
        let title = self.node_title_map.get(&hovered_id)
            .map(|t| t.as_str())
            .unwrap_or(&hovered_id);

        egui::show_tooltip_at_pointer(ctx, egui::Id::new("node_tooltip"), |ui| {
            if let Some(ann) = self.annotations.get(&hovered_id) {
                // Paper title (from node_title_map) + paper type badge
                ui.strong(format!("{} [{}]", title, &ann.paper_type));
                // Top 5 TF-IDF keywords
                if let Some(analysis) = self.analyses.get(&hovered_id) {
                    let kw_count = analysis.top_terms.len().min(5);
                    if kw_count > 0 {
                        ui.label(format!("Keywords: {}", analysis.top_terms[..kw_count].join(", ")));
                    }
                }
                // Primary method (name + category)
                if let Some(method) = ann.methods.first() {
                    ui.label(format!("Method: {} ({})", method.name, method.category));
                }
            } else {
                // Unanalyzed paper: show title + "Not analyzed"
                ui.strong(title);
                ui.label("Not analyzed");
            }
        });
    }
}
```

Note: `show_tooltip_at_pointer` signature in egui 0.31 may require `egui::Id` and a closure — check the exact API. Some versions use `(ctx, layer_id, id, |ui| {...})` and others use `(ctx, id, |ui| {...})`. Let the compiler guide you.
