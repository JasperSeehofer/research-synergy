    Rewrite the `run()` function in crawl.rs to use the queue-driven pattern. The function should:

    1. **Connect to DB** (same as before).

    2. **Run crash recovery:** Call `queue_repo.reset_stale_fetching().await?` and log the count of reset entries.

    3. **Check for existing queue entries:**
       - If `queue_repo.pending_count() > 0`, log "Resuming crawl from existing queue" and skip seed enqueue.
       - Otherwise, enqueue the seed paper: `queue_repo.enqueue_if_absent(&args.paper_id, &args.paper_id, 0)`.

    4. **Create shared resources:**
       - Rate limiter: `make_arxiv_limiter()` or `make_inspirehep_limiter()` based on `args.source`.
       - Concurrency: `let concurrency = args.parallel.unwrap_or(4);` — default 4 per user decision (CONTEXT.md: "Auto-tuned concurrency: starts at 4 tasks").
       - Semaphore: `Arc::new(Semaphore::new(concurrency))`
       - Progress broadcast channel: `let (progress_tx, _) = tokio::sync::broadcast::channel::<ProgressEvent>(256);` (ProgressEvent will be used by Plan 03; define a simple struct now with fields: papers_found: u64, papers_pending: u64, papers_failed: u64, current_depth: usize, max_depth: usize, elapsed_secs: f64, current_paper_id: Option<String>, current_paper_title: Option<String>, event_type: String).
       - Start time: `Instant::now()`
       - Counters: `Arc<AtomicU64>` for papers_found and papers_failed.

    5. **Create PaperSource factory:** Since `PaperSource` requires `&mut self` and is not Clone/Send-safe for sharing across tasks, each spawned task must create its own `PaperSource` instance. Create a factory closure or function:
       ```rust
       fn make_source(source_name: &str) -> Box<dyn PaperSource> { ... }
       ```
       For arXiv: create new `ArxivHTMLDownloader::new(create_http_client())` with `with_rate_limit(Duration::ZERO)` (rate limiting is now handled by governor, not per-source). For InspireHEP: `InspireHepClient::new(create_http_client())` also with zero rate limit.

    6. **Main crawl loop:**
       ```rust
       loop {
           let entry = queue_repo.claim_next_pending().await?;
           let Some(entry) = entry else {
               // No more pending — wait for in-flight tasks, then check again
               // If still nothing pending after all tasks complete, break
               break;
           };

           // Skip if beyond max_depth
           if entry.depth_level > args.max_depth {
               queue_repo.mark_done(&entry.paper_id, &entry.seed_paper_id).await?;
               continue;
           }

           // Skip if paper already in DB (check paper table, not just queue)
           if paper_repo.paper_exists(&entry.paper_id).await.unwrap_or(false) {
               // Still enqueue its references? No — if paper is in DB, its refs were already enqueued in a previous run.
               queue_repo.mark_done(&entry.paper_id, &entry.seed_paper_id).await?;
               // Update counters
               continue;
           }

           let permit = Arc::clone(&sem).acquire_owned().await.unwrap();
           let db_clone = db.clone();
           let limiter = Arc::clone(&rate_limiter);
           let tx = progress_tx.clone();
           let seed_id = args.paper_id.clone();
           let source_name = args.source.clone();
           let found_counter = Arc::clone(&papers_found);
           let failed_counter = Arc::clone(&papers_failed);

           tokio::spawn(async move {
               let _permit = permit;
               let mut source = make_source(&source_name);
               let queue = CrawlQueueRepository::new(&db_clone);
               let paper_repo = PaperRepository::new(&db_clone);

               // Rate limit
               wait_for_token(&limiter).await;

               // Fetch paper
               match source.fetch_paper(&entry.paper_id).await {
                   Ok(paper) => {
                       // Upsert paper to DB
                       let _ = paper_repo.upsert_paper(&paper).await;
                       let _ = paper_repo.upsert_citations(&paper).await;

                       // Fetch references and enqueue ALL (regardless of max_depth)
                       if let Ok(refs) = source.fetch_references(&entry.paper_id).await {
                           for r in &refs {
                               if let Some(ref arxiv_id) = r.arxiv_id {
                                   let _ = queue.enqueue_if_absent(arxiv_id, &seed_id, entry.depth_level + 1).await;
                               }
                           }
                       }

                       found_counter.fetch_add(1, Ordering::Relaxed);
                       queue.mark_done(&entry.paper_id, &seed_id).await.ok();

                       // Send progress event (ignore errors — no receivers is fine)
                       let _ = tx.send(ProgressEvent { event_type: "paper_fetched".to_string(), ... });
                   }
                   Err(e) => {
                       tracing::warn!(paper_id = entry.paper_id, error = %e, "Failed to fetch paper");
                       failed_counter.fetch_add(1, Ordering::Relaxed);
                       queue.mark_failed(&entry.paper_id, &seed_id).await.ok();
                       let _ = tx.send(ProgressEvent { event_type: "paper_failed".to_string(), ... });
                   }
               }
           });
       }
       ```

    7. **Wait for in-flight tasks:** After the loop breaks, wait for all spawned tasks to complete. Use a `JoinSet` or acquire all semaphore permits (acquire `concurrency` permits means all tasks have released theirs).

    8. **Retry failed entries (one retry per CONTEXT.md):** After all tasks complete, check if any entries are `failed` with `retry_count < 2`. If so, mark them pending via `queue_repo.retry_failed()` with a `WHERE retry_count < 2` filter, and re-enter the loop. Only do this once (one automatic retry on resume). Track this with a `retried: bool` flag.

    9. **Log final stats:** papers found, papers failed, elapsed time.

    10. **Run analysis if --analyze:** Keep the existing `run_analysis_pipeline` call, passing hardcoded `3` for rate_limit_secs.

    - `PaperSource` is NOT Send+Sync — each task must create its own instance via the factory.

    - The broadcast `tx.send()` should NOT unwrap — if no SSE client is connected, there are no receivers and that is fine.
