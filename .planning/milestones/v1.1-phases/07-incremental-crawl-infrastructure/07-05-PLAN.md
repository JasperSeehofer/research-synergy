Fix `CrawlSubcommand` so `--db` is accepted after the subcommand name. The clap parser currently rejects `crawl status --db <val>` because the unit variants define no arguments — once clap enters the subcommand context, `--db` has no parser to match against.

Purpose: UAT Test 2 and all dependent tests (3, 4, 5) are blocked by this. The fix is mechanical: convert unit variants to named struct variants each carrying their own `db: String` field, then read `db` from the variant in the dispatch block.

Output: Updated `crawl.rs` with struct CrawlSubcommand variants. No new deps required.

Root cause (from debug session):
- clap stops parsing CrawlArgs fields the moment it recognises a subcommand token
- Everything after `status` (including `--db`) goes to the Status variant's parser
- Unit variants have no fields → clap rejects `--db` as "unexpected argument"
- Fix: give each variant its own `--db` arg so clap can parse it in subcommand context

**Step 2: Update the subcmd dispatch block (lines 114-147)**

The dispatch block currently reads `args.db` for the connection string. Each variant now carries its own `db`. Update the match to destructure and use each variant's `db`:

```rust
if let Some(subcmd) = &args.subcmd {
    let db_str = match subcmd {
        CrawlSubcommand::Status { db } => db,
        CrawlSubcommand::Clear { db } => db,
        CrawlSubcommand::Retry { db } => db,
    };
    let db = match resyn_core::database::client::connect(db_str).await {
        Ok(db) => {
            info!(endpoint = db_str.as_str(), "Connected to database");
            db
        }
        Err(e) => {
            error!(error = %e, "Failed to connect to database");
            std::process::exit(1);
        }
    };
    let queue = CrawlQueueRepository::new(&db);
    match subcmd {
        CrawlSubcommand::Status { .. } => {
            let counts = queue.get_counts().await?;
            println!("Crawl Queue Status:");
            println!("  Total:    {}", counts.total);
            println!("  Pending:  {}", counts.pending);
            println!("  Fetching: {}", counts.fetching);
            println!("  Done:     {}", counts.done);
            println!("  Failed:   {}", counts.failed);
        }
        CrawlSubcommand::Clear { .. } => {
            queue.clear_queue().await?;
            println!("Crawl queue cleared.");
        }
        CrawlSubcommand::Retry { .. } => {
            let count = queue.retry_failed().await?;
            println!("Marked {count} failed entries as pending for retry.");
        }
    }
    return Ok(());
}
```

Note: alternatively, collapse the two match blocks into one by extracting db_str and the queue operation in a single pass — either approach is acceptable as long as the final behavior is correct.
