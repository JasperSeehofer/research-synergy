# Phase 8 — UI Design Contract

> Visual and interaction contract for the Leptos CSR web shell and analysis panels.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none — plain CSS via Trunk | CONTEXT.md: "Separate CSS file(s) loaded by Trunk — no inline styles, no CSS-in-Rust" |
| Preset | not applicable | Rust/WASM project — shadcn is React-only |
| Component library | none — hand-rolled Leptos components | RESEARCH.md: pattern library is leptos + leptos_router |
| Icon library | none in this phase — text labels only | Graph icons deferred to Phase 9 |
| Font | System UI stack: -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif | Default — no custom font load in Phase 8 |

**Note:** This is a Rust/Leptos/WASM project. shadcn, Tailwind, and React component libraries are not applicable. All styling is hand-authored in `resyn-app/style/main.css` and linked via `index.html` Trunk directives.

## Layout Architecture

Declared layout regions (from CONTEXT.md):

| Region | Behavior | CSS Approach |
|--------|----------|--------------|
| Sidebar (expanded) | 240px fixed width, full height, collapsible | `width: 240px; transition: width 200ms ease` |
| Sidebar (collapsed / rail) | 48px icon-only strip with tooltips on hover | `width: 48px` — Leptos `RwSignal<bool>` drives class swap |
| Content area | Fills remaining viewport width | `flex: 1; overflow-y: auto` |
| Paper detail drawer | 480px, slides in from right, overlays content | `position: fixed; right: 0; transform: translateX(0/100%)` |
| Crawl progress (sidebar bottom) | Always visible in sidebar footer | `margin-top: auto` pinned to bottom |
| App shell | Full-viewport flex row | `display: flex; height: 100vh; overflow: hidden` |

Sidebar collapse state is a `RwSignal<bool>` in the Leptos reactive graph — never a JS class toggle. Class attribute uses `move || if collapsed.get() { "sidebar rail" } else { "sidebar expanded" }`.

## Spacing Scale

Declared values (all multiples of 4px):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, inline tag padding, confidence bar inner padding |
| sm | 8px | Card inner padding (compact), table cell vertical padding, badge padding |
| md | 16px | Default card padding, sidebar nav item padding, form field spacing |
| lg | 24px | Section header margin-bottom, dashboard card gap |
| xl | 32px | Panel content padding (horizontal), drawer content padding |
| 2xl | 48px | Sidebar rail width (icon-only mode), major section breaks |
| 3xl | 64px | Dashboard card minimum width reference |

Exceptions:
- Sidebar rail collapsed width: 48px (matches 2xl token for consistency)
- Sidebar expanded width: 240px (not a scale token — fixed layout value)
- Paper detail drawer width: 480px (not a scale token — fixed layout value)
- Touch target minimum for nav items: 44px height (accessibility)

## Typography

Font: System UI stack. No web font load — reduces latency for a local research tool.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Table cells, card body text, drawer abstract text |
| Label | 12px | 600 (semibold) | 1.3 | Column headers, badge text, nav item labels, form labels, sidebar section headings |
| Heading | 18px | 600 (semibold) | 1.3 | Panel page titles, drawer paper title |

Weights used: exactly 2 — 400 (regular) and 600 (semibold).

## Color

Dark minimal theme (VS Code / Linear aesthetic). Source: CONTEXT.md decision.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0d1117` | App background, main content area background, drawer background |
| Secondary (30%) | `#161b22` | Sidebar background, cards, table row hover, heatmap container |
| Surface-raised | `#1c2128` | Card background, table header row, modal overlay |
| Accent (10%) | `#58a6ff` | Active sidebar nav item, primary CTA button, confidence bar fill (high confidence end), column sort indicator |
| Accent-muted | `#1f3a5f` | Accent element hover state, selected row background |
| Contradiction badge | `#f85149` | Contradiction type badge background only |
| Bridge badge | `#d29922` | ABC-Bridge type badge background only |
| Success / high-confidence | `#3fb950` | Confidence bar at 80–100% fill color |
| Warning / mid-confidence | `#d29922` | Confidence bar at 40–79% fill color |
| Destructive | `#f85149` | Destructive action buttons only (clear queue confirmation) |
| Border | `#30363d` | Card borders, table borders, separator lines, sidebar divider |
| Text-primary | `#e6edf3` | All body and heading text |
| Text-muted | `#8b949e` | Secondary labels, timestamps, muted metadata, placeholder text |
| Text-inverse | `#0d1117` | Text on accent/badge colored backgrounds |

Accent (`#58a6ff`) reserved for: active nav item highlight, primary CTA "Start Crawl" button, confidence bar high-value fill, sort-active column header indicator. It is NOT used on every interactive element.

Confidence bar color uses a linear gradient, not discrete classes: `#f85149` (0%) → `#d29922` (50%) → `#3fb950` (100%)`. The numeric value is displayed as a percentage label beside the bar (e.g., "73%").

Method heatmap cell colors (sequential blue scale, source: CONTEXT.md):
- Empty cell (0 papers): `#1c2128` background + `#30363d` dashed border
- Low (1 paper): `#1e3a5f`
- Medium (2–5 papers): `#1a5fa0`
- High (6+ papers): `#00cfff`

## Component Inventory

Components that must exist in `resyn-app/src/components/` by end of Phase 8:

### layout/sidebar.rs
- Expanded state: 240px, shows nav labels + crawl progress footer
- Collapsed state: 48px rail, icon-only, tooltip on hover (CSS `:hover + .tooltip`)
- Nav items: Dashboard, Papers, Gaps, Open Problems, Methods — each links to a route
- Active nav item: left border accent stripe (`border-left: 3px solid #58a6ff`) + accent text color
- Collapse toggle: chevron icon button at sidebar top-right

### layout/drawer.rs
- Slides in from right: `transform: translateX(100%)` → `translateX(0)`, `transition: transform 200ms ease`
- Width: 480px, fixed position, full height, `z-index: 100`
- Backdrop: `rgba(0,0,0,0.5)` behind drawer, click-to-close
- Sections: paper title (Heading), authors + year (Label/muted), abstract (Body), methods list, findings list, open problems list
- Close button: top-right X, `24×24px` touch target

### components/gap_card.rs
- Container: card background `#1c2128`, border `#30363d`, border-radius `6px`, padding `md (16px)`
- Type badge: pill shape, text Label (12px semibold), `#f85149` bg for Contradiction / `#d29922` bg for Bridge, `#0d1117` text
- Paper title links: Body (14px), accent color `#58a6ff`, cursor pointer, opens paper drawer on click
- Shared terms: pill tags, `#161b22` bg, border `#30363d`, Label (12px), padding `xs sm (4px 8px)`
- Confidence bar: full-width horizontal bar, `8px` height, `border-radius: 4px`, gradient fill, percentage label right-aligned beside bar
- Expand/collapse for justification: chevron toggle, justification text Body (14px) muted color
- Card gap between cards: `lg (24px)`

### components/heatmap.rs
- CSS grid layout, N×N where N = number of method categories (capped at top 10 by paper count)
- Axis labels: Label (12px semibold), rotated 45° for column headers
- Cell: square, minimum `40×40px`, border-radius `3px`, cursor pointer on non-empty cells
- Cell color per scheme defined in Color section above
- Empty cells: dashed border, no hover effect
- Non-empty cells: `opacity: 0.8` on hover, pointer cursor, tooltip showing count and category pair
- On click: drill-down renders a secondary N×M grid of individual method names for the two selected categories

### components/crawl_progress.rs
- Expanded: progress bar (`8px` height, accent fill), stats text (Label 12px muted), current paper title (Body 14px, truncated with ellipsis at 200px)
- Collapsed (rail): `20px` spinning indicator (CSS `@keyframes` spin), percentage text (Label 12px) below it
- Idle state: "Last crawl: N papers in Xm Ys" summary text (Label 12px muted)
- Crawl form (expanded): `<input>` for paper ID, `<select>` for depth (1–10), `<select>` for source (arxiv/inspirehep), "Start Crawl" button (primary accent)

### pages/dashboard.rs
- 5 summary cards in a flex-wrap row, gap `lg (24px)`, minimum card width `160px`
- Card: title Label (12px semibold muted), number Display (28px regular accent), link text Label (12px) muted — "View all →"
- Cards: Total Papers, Contradictions, ABC-Bridges, Open Problems, Method Coverage %

### pages/papers.rs
- Full-width `<table>`, `border-collapse: collapse`
- Columns: Title (flex: 3), Authors (flex: 2), Year (fixed 60px), Citations (fixed 80px), Status (fixed 120px)
- Column header: Label (12px semibold), sortable — sort icon appears on hover, accent color when active sort
- Row: Body (14px), `border-bottom: 1px solid #30363d`, hover background `#1f3a5f`
- Row click: opens paper detail drawer
- Status badge: "Analyzed" (`#3fb950` text) / "Pending" (`#8b949e` text) — text-only, no background

### pages/gaps.rs
- Filter bar at top: "Contradictions" toggle button, "Bridges" toggle button, confidence threshold slider (0–100%)
- Toggle active state: accent background, text-inverse; inactive: surface-raised background, text-muted
- Card list below filter bar, gap-card component per finding
- Empty state: centered, muted text (see Copywriting section)

### pages/open_problems.rs
- Ranked list with visible rank numbers (`#1`, `#2`, …)
- Each row: rank (Label 12px muted, fixed 32px), problem text (Body 14px), recurrence count badge (Label 12px, accent background, right-aligned)
- Row separator: `border-bottom: 1px solid #30363d`
- Row hover: `#161b22` background

### pages/methods.rs
- Panel title + description text above heatmap
- Heatmap component centered in content area
- Drill-down sub-matrix appears below primary heatmap when a cell is clicked, with a "Back to overview" link (Label 12px, accent color)

## Interaction Contracts

### Sidebar collapse/expand
- Trigger: chevron button click (top of sidebar)
- Animation: `width` transition `200ms ease` — no jarring layout shift
- Icon tooltip: appears on `200ms` hover delay, position right of icon, max-width `180px`, `#161b22` bg, `#e6edf3` text, `border-radius: 4px`, `padding: 4px 8px`
- Reactive: `RwSignal<bool>` in Leptos — child components re-render via signal read

### Paper drawer open/close
- Open trigger: row click in Papers table OR paper link click in Gap card
- Animation: `transform: translateX` transition `200ms ease`
- Close trigger: backdrop click OR X button click
- State: `RwSignal<Option<String>>` holding selected paper ID; `None` = drawer closed

### Gap card expand
- Trigger: chevron icon click (bottom-right of card)
- Behavior: justification text slides in below shared terms — CSS `max-height` transition `150ms ease`
- Collapsed: `max-height: 0; overflow: hidden`; Expanded: `max-height: 300px`

### Table column sort
- Trigger: column header click
- Behavior: sorted client-side (no server refetch); sort indicator arrow rotates 180° for desc
- State: `RwSignal<(SortColumn, SortDirection)>` in Leptos

### Method heatmap drill-down
- Trigger: click on non-empty heatmap cell
- Behavior: sub-matrix renders below primary heatmap in the same panel (no navigation change)
- State: `RwSignal<Option<(String, String)>>` holding selected category pair; `None` = no drill-down shown
- "Back to overview": clears the signal, sub-matrix disappears

### Confidence slider
- Component: `<input type="range" min="0" max="100" step="5">`
- Label shows current value live: "Min confidence: 45%"
- Gap cards below filtered reactively via derived signal

### Start Crawl form
- Validation: paper ID field must match arXiv format before submit button enables
- On submit: calls `start_crawl()` server function via Leptos `Action`
- Loading state: button text changes to "Starting…", disabled, no spinner needed
- On success: SSE progress stream activates (leptos-use `use_event_source` connects to `/progress`)
- On error: error message below form (see Copywriting)

## Loading and Error States

Loading states use skeleton placeholders (CSS animated shimmer), not spinners, for data panels. Source: CONTEXT.md ("Loading states and skeleton screens" in Claude's Discretion).

| Context | Loading State | Error State |
|---------|---------------|-------------|
| Papers table | 8 skeleton rows, same column structure, shimmer animation | Error banner above table (see Copywriting) |
| Gap cards | 3 skeleton cards, same height as real cards | Error banner in panel |
| Open problems list | 5 skeleton rows | Error banner in panel |
| Method heatmap | Gray placeholder grid same size as expected | Error banner in panel |
| Dashboard cards | Numbers replaced with "—", shimmer on number area | Card shows "—" with "Failed to load" label below |
| Paper drawer | Skeleton for title + abstract area while loading | Inline error in drawer |

Skeleton shimmer: `background: linear-gradient(90deg, #1c2128 25%, #21272f 50%, #1c2128 75%)`, `background-size: 200% 100%`, `animation: shimmer 1.5s infinite`.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigation | All interactive elements reachable via Tab; drawer traps focus while open |
| ARIA roles | `role="navigation"` on sidebar, `role="dialog"` on drawer, `aria-expanded` on sidebar toggle |
| Color contrast | Text-primary `#e6edf3` on dominant `#0d1117`: contrast ratio ~14:1 (WCAG AAA). Text-muted `#8b949e` on dominant: ~5:1 (WCAG AA) |
| Sort state | `aria-sort="ascending"/"descending"` on `<th>` elements |
| Badge text | Badge text `#0d1117` on `#f85149` (contradiction): ~4.9:1 contrast — WCAG AA |
| Drawer | `aria-modal="true"`, focus returns to trigger element on close |
| Heatmap | Each cell has `aria-label="{row} × {col}: {N} papers"` |
