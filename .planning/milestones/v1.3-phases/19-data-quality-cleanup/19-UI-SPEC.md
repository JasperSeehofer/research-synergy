## Phase UI Classification

**Classification: Backend-only fix with observable frontend side effects.**

Phase 19 modifies two Rust data layer files (`inspirehep_api.rs`, `paper.rs`). No new UI components are introduced. No existing UI components are modified. The executor writes only Rust code in `resyn-core`. The observable frontend consequences are:

1. The temporal year-range slider (existing `TemporalSlider` component, `graph_controls.rs`) will show non-empty year bounds after an InspireHEP crawl — previously all `published` fields were `""`, so the slider showed nothing to filter.
2. The graph (existing WebGL renderer, `webgl_renderer.rs`) will contain no orphan nodes — previously empty-ID references created disconnected nodes with no edges.

The design contract below exists to constrain those observable states and ensure the executor does not introduce new UI patterns or deviate from the established design system.

## Interaction Contract

This section defines the observable interaction behaviour that must hold after Phase 19 is complete. These are acceptance criteria expressed as UI behaviour.

### Temporal Slider — Year Bounds (ARXIV-02)

- After an InspireHEP depth-2+ crawl, `temporal_min_year` and `temporal_max_year` in `layout_state.rs` must be computed from non-empty year strings.
- The year-range slider must show a valid range (e.g. "2019 – 2025"), not a degenerate range (e.g. "0 – 0" or a single year).
- Dragging either thumb filters nodes in/out of the graph — this is the existing interaction, unchanged.
- Year string format: 4-character slice from `published` field (e.g. `"2025"` from `"2025-03-24"`). This matches the existing extraction in `server_fns/graph.rs` — the executor must not change the extraction logic.

### Graph Orphan State — Zero Isolated Nodes (ORPH-02)

- After an InspireHEP depth-2+ crawl, no node in the WebGL graph must be rendered without at least one visible edge.
- The node count indicator reads "Showing {n} of {m} nodes" — `n` and `m` must be equal before any temporal filtering is applied (all nodes are connected, none are hidden as orphans).
- The executor must not add any post-render orphan detection, graph traversal, or node visibility filtering to achieve this. The fix lives entirely in `get_arxiv_references_ids()`.
