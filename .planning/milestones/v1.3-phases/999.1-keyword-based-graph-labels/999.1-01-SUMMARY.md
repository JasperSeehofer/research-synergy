# Phase 999.1 Plan 01: Extend Data Pipeline for Keyword Labels Summary

TF-IDF keyword data pipeline from AnalysisRepository to WASM frontend via GraphNode and NodeState, plus LabelMode enum foundation for rendering decisions.

## What Was Built

- **GraphNode.top_keywords**: `Vec<(String, f32)>` field with `#[serde(default)]` for backward-compatible JSON deserialization. Populated by zipping `PaperAnalysis.top_terms` + `top_scores` (capped at 5).
- **Server function join**: `get_graph_data()` SSR block now calls `AnalysisRepository::new(&db).get_all_analyses()` once, builds a `HashMap<String, Vec<(String, f32)>>` keyed by `arxiv_id`, then populates each `GraphNode.top_keywords` via `analysis_map.get(&p.id).cloned().unwrap_or_default()`.
- **LabelMode enum**: `AuthorYear` (default), `Keywords`, `Off` variants. Uses `#[derive(Default)]` with `#[default]` attribute.
- **NodeState.top_keywords**: `Vec<(String, f32)>` field, populated from `GraphNode.top_keywords` in `from_graph_data()`.
- **GraphState.label_mode**: `LabelMode` field initialized to `LabelMode::default()` (AuthorYear).
- **Test helpers updated**: `make_node`, `make_node_with_depth`, `make_node_with_year` in layout_state.rs; `make_test_node` in label_collision.rs; `make_node` and 3 inline NodeState constructions in lod.rs — all include `top_keywords: vec![]`.
- **New tests added**: `test_graph_node_top_keywords_serde_default`, `test_graph_node_top_keywords_round_trip`, `test_label_mode_default_is_author_year`, `test_from_graph_data_populates_top_keywords`.
