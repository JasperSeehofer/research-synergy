**Why human:** Requires a live WASM environment with real analyzed paper data. Cannot be tested programmatically or with `cargo test`.

#### 2. Cluster Convex Hull Borders

**Test:** In Keywords mode, zoom out until viewport scale drops below 0.6.
**Expected:** Dashed blue-tinted convex hull borders (`rgba(88, 166, 255, 0.25)`) appear around node clusters. Cluster label pills show the dominant keyword at each cluster centroid.
**Why human:** Requires browser interaction (zoom gesture) and enough nodes to trigger meaningful cluster formation.

#### 3. LOD Cluster Transition

**Test:** From the zoomed-out state above, zoom back in past scale 0.6.
**Expected:** Cluster hulls and centroid labels disappear cleanly. Per-node keyword pills take over.
**Why human:** Requires continuous zoom interaction to observe the transition point.

#### 4. Extended Tooltip in Keywords Mode

**Test:** In Keywords mode, hover over an analyzed node, then hover over an unanalyzed node.
**Expected:** Analyzed node tooltip shows separator line, "Keywords:" heading, and up to 5 keyword/score pairs. Unanalyzed node tooltip shows only title/author/citations with no keywords section.
**Why human:** Requires live app with mixed analyzed/unanalyzed nodes.
