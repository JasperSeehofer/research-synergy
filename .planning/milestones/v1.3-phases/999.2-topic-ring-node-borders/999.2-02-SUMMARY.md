# Phase 999.2 Plan 02: Canvas2D Topic Arc Rendering and UI Controls Summary

Canvas2D topic ring rendering pipeline with colored arc segments encoding top-3 TF-IDF keywords on node borders, dashed ring for unanalyzed nodes, Topic Rings toggle, Topic Colors legend with click-to-filter dimming in the graph controls sidebar.

## What Was Built

**compute_arc_angles() pure function** — Takes top-keywords slice and palette slot map, returns `Vec<(f64, f64, Option<u8>)>` arc segments. Each raw TF-IDF score treated as a fraction of the full 2π circle. Neutral remainder arc appended when sum < 1.0. Starts at -PI/2 (12 o'clock). 6 unit tests.

**draw_topic_rings()** — Iterates all nodes, skips non-visible and below-threshold (< 6px screen radius). Builds palette lookup maps, calls `compute_arc_angles()`, strokes each arc with the palette color or neutral `#3a424c`. Applies `dim_alpha = 0.3` when `node.topic_dimmed`.

**draw_dashed_ring()** — Strokes a 4px-on/4px-off dashed neutral ring (`#6e7681`) for unanalyzed nodes. Resets dash pattern before `ctx.restore()` to avoid leaking.

**RAF integration** — Topic rings drawn on the overlay canvas at all zoom levels, before the label drawing block. `show_topic_rings.get_untracked()` gate. Topic_dimmed computed per-frame from `active_topic_filter`.

**canvas_renderer.rs is_dimmed extension** — OR-combines selection-based dimming with `node.topic_dimmed` so node fills also dim when topic filter active.

**GraphControls extension** — Three new params: `show_topic_rings`, `active_topic_filter`, `palette`. Topic Rings toggle button (default on). TOPIC COLORS legend section conditionally rendered when rings on and palette non-empty. Legend entries are clickable buttons with colored swatches; click toggles keyword in `active_topic_filter` HashSet (union semantics).

**CSS** — `.topic-legend-section`, `.topic-legend-entries` (scrollable max 160px), `.legend-entry`, `.legend-entry.active` (`#1f3a5f`), `.legend-swatch` (12px × 12px).

## Deviations from Plan

**[Rule 1 - Bug] Fixed compute_arc_angles normalization logic**
- **Found during:** Task 1 (test failure)
- **Issue:** Initial implementation normalized scores by dividing by `raw_sum`, which caused sum=0.5 to fill the whole circle — no remainder arc would be appended, breaking `test_arc_angles_two_keywords_with_remainder`
- **Fix:** Changed to treat each raw TF-IDF score directly as a circle fraction (clamped to [0,1]). If sum < 1.0, remainder arc naturally appears. Matches the spec: "Remaining circumference renders in neutral color when TF-IDF scores sum < 1.0"
- **Files modified:** resyn-app/src/pages/graph.rs
- **Commit:** 99b71a8
