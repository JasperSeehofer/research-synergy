    3. Create `src/nlp/preprocessing.rs`:
    - `pub fn tokenize(text: &str) -> Vec<String>` -- split on non-alphanumeric chars, filter len > 2, lowercase. Simple whitespace+punctuation tokenizer per RESEARCH.md.
    - `pub fn build_stop_words() -> HashSet<String>` -- use `stop_words::get(LANGUAGE::English)` plus hardcoded academic boilerplate: "paper", "study", "result", "show", "figure", "also", "using" (locked decision from CONTEXT.md).

    - `pub fn compute_weighted_tf(sections: &SectionMap, stop_words: &HashSet<String>) -> HashMap<String, f32>` -- section weights: abstract 2.0, methods 1.5, results 1.0, introduction 0.5, conclusion 0.5 (locked decision). Use float weight multiplication at TF stage (not token duplication) per RESEARCH.md recommended approach. For each section, compute per-token TF as weight/n_tokens, accumulate across sections, then normalize by total weight sum.
    - `pub fn compute_smooth_idf(doc_freq: usize, total_docs: usize) -> f32` -- formula: `ln((1+N)/(1+df)) + 1` (per RESEARCH.md Pattern 2).
    - `pub fn corpus_fingerprint(arxiv_ids: &[String]) -> String` -- sort IDs, join with comma, SHA-256 hash via `sha2` crate. Returns hex string.

    - `pub struct TfIdfEngine` with methods:

      - `pub fn compute_corpus(extractions: &[TextExtractionResult]) -> Vec<(String, HashMap<String, f32>)>` -- returns (arxiv_id, tfidf_vector) pairs. Steps:

        b. Compute weighted TF for each document
        c. Build document-frequency map across all documents
        d. Compute IDF for each term (corpus-level, done ONCE -- per RESEARCH.md anti-pattern warning)
        e. Apply TF * IDF for each document
      - `pub fn get_top_n(tfidf: &HashMap<String, f32>, n: usize) -> (Vec<String>, Vec<f32>)` -- returns parallel arrays (top_terms, top_scores) sorted by descending score.

    4. **Extract top-5 and persist**: For each (arxiv_id, tfidf_vector):

       - Call `analysis_repo.upsert_analysis(&analysis).await`
