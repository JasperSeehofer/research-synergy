  <action>
    1. Extend `LlmProvider` trait in `src/llm/traits.rs` with a new method:
       ```rust
       async fn verify_gap(
           &mut self,
           prompt: &str,
           context: &str,
       ) -> Result<String, ResynError>;
       ```
       This returns the raw justification string (or "NO" if not a real gap). Keeps gap verification separate from structured annotation. Per RESEARCH.md: no JSON parsing needed — simple string response.

    2. Create `src/llm/gap_prompt.rs` with two pub const prompt templates:
       - `CONTRADICTION_SYSTEM_PROMPT`: Instructs the LLM to compare two paper summaries and their findings, determine if they genuinely contradict each other on the same topic. If yes, respond with a 1-2 sentence justification explaining the contradiction. If no, respond with exactly "NO".
       - `ABC_BRIDGE_SYSTEM_PROMPT`: Instructs the LLM to examine two papers (A and C) that don't directly cite each other but share intermediary keywords via B papers. Generate a 1-2 sentence justification explaining how A's work connects to C's work through the shared concepts. If no meaningful connection, respond with "NO".
       - Add `pub mod gap_prompt;` to `src/llm/mod.rs`

    3. Implement `verify_gap` on all three existing providers:
       - `ClaudeProvider`: Reuse internal `rate_limit_check()` and `call_api()` — call with the provided prompt as system message and context as user content. Return the response body text directly. Follow existing `annotate_paper` pattern but skip JSON parsing.
       - `OllamaProvider`: Same approach — reuse `rate_limit_check()` and `call_api()` with prompt/context. Return raw text response.
       - `NoopProvider`: Return `"NO".to_string()` (noop never confirms gaps, consistent with producing empty-but-valid results).

    4. Verify all existing tests still pass — the new trait method must not break existing `annotate_paper` implementations.
