Define all server functions using the Leptos 0.8 `#[server]` macro. Each function is defined in resyn-app but the body only compiles server-side (behind `#[cfg(feature = "ssr")]`).

**CRITICAL pattern for all server functions:**
- Function signature uses only WASM-safe types (types from resyn-core without ssr feature, or serde-serializable wrapper types defined here)
- Function body is gated: `#[cfg(feature = "ssr")] { ... actual DB logic ... }` and `#[cfg(not(feature = "ssr"))] { unreachable!() }`
- DB handle retrieved via: `use_context::<std::sync::Arc<resyn_core::database::client::Db>>().ok_or_else(|| ServerFnError::new("Database not available"))?`
- All ssr-only imports (database modules) inside the `#[cfg(feature = "ssr")]` block

**resyn-app/src/server_fns/papers.rs:**

1. `get_papers()` — returns `Result<Vec<Paper>, ServerFnError>`. Calls `PaperRepository::new(&db).get_all_papers().await`. Map `ResynError` to `ServerFnError` via `.map_err(|e| ServerFnError::new(e.to_string()))`.

2. `get_paper_detail(id: String)` — returns `Result<PaperDetail, ServerFnError>` where `PaperDetail` is a custom struct (Serialize + Deserialize) containing:
```rust
#[derive(Clone, Debug, serde::Serialize, serde::Deserialize)]
pub struct PaperDetail {
    pub paper: Paper,
    pub annotation: Option<LlmAnnotation>,
}
```
Calls `PaperRepository::get_paper(&id)` and `LlmAnnotationRepository::get_annotation(&id)` (check the exact method signature — it might be `get_annotation(&self, arxiv_id: &str)`).

3. `get_dashboard_stats()` — returns `Result<DashboardStats, ServerFnError>` where:
```rust
#[derive(Clone, Debug, serde::Serialize, serde::Deserialize)]
pub struct DashboardStats {
    pub total_papers: usize,
    pub contradiction_count: usize,
    pub bridge_count: usize,
    pub open_problems_count: usize,
    pub method_coverage_pct: f32,
}
```
Queries: `get_all_papers().len()`, counts from `get_all_gap_findings()` filtered by `GapType::Contradiction` and `GapType::AbcBridge`, unique open_problems count from `get_all_annotations()`, and method_coverage_pct = (number of non-zero cells in the method category pair matrix / total possible pairs * 100).

4. `start_crawl(paper_id: String, max_depth: usize, source: String)` — returns `Result<String, ServerFnError>`. For now, this is a placeholder that returns `Ok("Crawl started.".to_string())` — the actual crawl triggering logic will be wired in Plan 04. Add a TODO comment.

1. `get_method_matrix()` — returns `Result<MethodMatrix, ServerFnError>` where:
```rust
#[derive(Clone, Debug, serde::Serialize, serde::Deserialize)]
pub struct MethodMatrix {

    pub cells: Vec<MatrixCell>,
}

#[derive(Clone, Debug, serde::Serialize, serde::Deserialize)]
pub struct MatrixCell {

```
Aggregation logic per RESEARCH.md: get all annotations, collect `Method.category` pairs per paper (both (A,B) and (B,A) normalized so row <= col alphabetically), count occurrences, return top 10 categories by total paper count, build cells for all pairs among those top 10.

2. `get_method_drilldown(cat_a: String, cat_b: String)` — returns `Result<MethodMatrix, ServerFnError>`. Same structure but uses individual method names (not categories) for the two selected categories. Filters annotations to only those having methods in cat_a or cat_b.
