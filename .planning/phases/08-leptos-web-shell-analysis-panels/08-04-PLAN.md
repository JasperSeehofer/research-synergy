**Heatmap component (resyn-app/src/components/heatmap.rs):**

Create a `Heatmap` component that accepts a `MethodMatrix` prop and an `on_cell_click` callback:

```rust
#[component]
pub fn Heatmap(
    matrix: MethodMatrix,
    #[prop(optional)] on_cell_click: Option<Callback<(String, String)>>,
) -> impl IntoView { ... }
```

1. **Grid layout**: Create a CSS grid with `grid-template-columns: auto repeat(N, 1fr)` and `grid-template-rows: auto repeat(N, 1fr)` where N = `matrix.categories.len()`. The first row/column are axis labels.

   Set the grid style dynamically: `style=format!("grid-template-columns: 80px repeat({}, minmax(40px, 1fr)); grid-template-rows: auto repeat({}, minmax(40px, 1fr));", n, n)`.

2. **Column header labels** (first row): For each category, render a `div class="heatmap-label"` with the category name. Rotate 45 degrees: add CSS style `transform: rotate(-45deg); transform-origin: left bottom;` or use a dedicated class. Place in the grid starting at column 2.

3. **Row labels + cells**: For each row category (index i):
   - First column: `div class="heatmap-label"` with row category name
   - For each column category (index j): Look up the cell count from `matrix.cells` where `(row, col)` matches the category pair (normalized alphabetically). Render a `div` with:
     - Class: `"heatmap-cell"` + cell color class based on count:
       - count == 0: `" cell-empty"`
       - count == 1: `" cell-low"`
       - count 2..=5: `" cell-medium"`
       - count >= 6: `" cell-high"`
     - `title` attribute: `"{row_category} x {col_category}: {count} papers"` per UI-SPEC copywriting
     - `aria-label`: same as title
     - Text content: show count if > 0, empty if 0
     - On click (if not empty and on_cell_click is Some): call the callback with `(row_category.clone(), col_category.clone())`

4. **Empty cells** (count == 0): dashed border, no hover effect, no cursor pointer, no click handler.

**CrawlProgress component (resyn-app/src/components/crawl_progress.rs):**

Create a `CrawlProgress` component that connects to the SSE endpoint and provides a crawl launcher form:

1. **SSE connection**: Use `leptos_use::use_event_source` to connect to `/progress`:
```rust
use leptos_use::{UseEventSourceReturn, use_event_source_with_options, UseEventSourceOptions};
use codee::string::FromToStringCodec; // or JsonSerdeCodec if available
```

The SSE endpoint sends JSON-encoded `ProgressEvent`. Use `use_event_source::<String, FromToStringCodec>("/progress")` and manually deserialize the message, or use `JsonSerdeCodec` with `codee` if it supports serde. The executor should check which codec approach works with `leptos-use 0.18` — the key requirement is to get `ProgressEvent` data from the SSE stream into a reactive signal.

Store the latest progress event in a local signal: `let latest_progress = RwSignal::new(Option::<ProgressEvent>::None)`. Update it when a new SSE message arrives.

2. **Crawl state detection**: Determine state from latest_progress:
   - `None` (never received an event) or last event has `event_type == "complete"`: **idle**
   - Event with `event_type == "paper_fetched"` or `"paper_failed"`: **running**

3. **Read sidebar_collapsed from context** to determine expanded vs rail rendering.

4. **Expanded mode** (sidebar not collapsed):
   - **When running**: Show:
     - Progress bar: `div class="progress-bar"` with inner `div class="progress-bar-fill"` width = `papers_found / (papers_found + papers_pending) * 100` percent. If `papers_pending == 0`, show 100%.
     - Stats text (class `"text-label text-muted"`): `"{papers_found} papers found · {papers_pending} pending · depth {current_depth}/{max_depth}"`
     - Elapsed time: format `elapsed_secs` as `"{minutes}m {seconds}s"`
     - Current paper title (class `"text-body"`, truncated with ellipsis via CSS `overflow: hidden; text-overflow: ellipsis; white-space: nowrap; max-width: 200px`): show `current_paper_title` or `current_paper_id` if title is None
   - **When idle**: Show summary text (class `"text-label text-muted"`):
     - If a previous event exists (complete): "Last crawl: {papers_found} papers in {elapsed_time}"
     - If no event ever received: "No crawl history"
   - **Crawl form** (always visible when expanded):
     - Paper ID input: `input type="text" placeholder="e.g. 2503.18887"` bound to a local `RwSignal<String>`
     - Max depth select: `select` with options 1 through 10, bound to `RwSignal<usize>`, default 3
     - Source select: `select` with options "arxiv" and "inspirehep", bound to `RwSignal<String>`, default "arxiv"
     - "Start Crawl" button: `button class="btn-primary"`, disabled when paper ID is empty. On click, call `start_crawl` server function via Leptos `Action`:
       ```rust
       let crawl_action = Action::new(move |_: &()| {
           let paper_id = paper_id_signal.get();
           let depth = depth_signal.get();
           let source = source_signal.get();
           async move { start_crawl(paper_id, depth, source).await }
       });
       ```
     - Loading state: button text changes to "Starting..." and is disabled while action is pending
     - On success: show "Crawl started. Progress updates will appear in the sidebar." as a brief message (disappears after 3 seconds or on next progress event)
     - On error: show error message below form: "Failed to start crawl: {error}. Check the paper ID format and server status."

5. **Collapsed mode** (sidebar rail):
   - **When running**: `div class="spinner"` (20px spinning indicator) + `span` showing percentage text below it
   - **When idle**: nothing or a small static icon
