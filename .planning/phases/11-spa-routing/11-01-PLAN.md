<objective>
Fix the Axum production server so all client-side routes work on direct URL navigation and browser refresh.

Purpose: Currently navigating to any route other than `/` returns 404 because `ServeDir` looks for a physical file matching the path. The Leptos client-side `<Router>` never gets a chance to handle the route.

Output: A one-line change to `serve.rs` that makes `ServeDir` fall back to `index.html` for any path that doesn't match a static file, while preserving `/progress` (SSE) and `/api/*` (server functions) routes.
</objective>
