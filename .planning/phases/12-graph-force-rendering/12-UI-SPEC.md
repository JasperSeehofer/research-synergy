## Graph Rendering Contract

This section is specific to Phase 12. It defines the visual outcome the fixed rendering pipeline must produce.

### Node Appearance

| Property | Value | Rationale |
|----------|-------|-----------|
| Node radius | 6px world-space (CSS units at scale=1.0) | Pre-existing shader — do not change |
| Node shape | Circle via SDF in vertex shader | Pre-existing — `smoothstep` edge produces crisp anti-aliasing |
| Edge smoothness | `smoothstep(0.0, 1.5/radius, dist)` | Pre-existing SDF formula — produces sub-pixel smooth edge at all DPR |
| Label visibility | Shown at LOD >= 1.0 (zoom > threshold) | Pre-existing LOD system — do not change threshold |

### DPR / Crispness Contract

The WebGL pipeline must satisfy all four conditions simultaneously. Failure on any one produces blur or coordinate mismatch.

| Layer | Required Value | Verification |
|-------|---------------|-------------|
| Canvas physical size | `css_width * dpr` × `css_height * dpr` (physical pixels) | `canvas.width()` after `set_width()` |
| GL viewport | `(0, 0, physical_w, physical_h)` — physical pixels | `gl.viewport()` call in `resize()` |
| Shader `u_resolution` | `(physical_w / dpr, physical_h / dpr)` = CSS pixels | Lines ~164-166 in `webgl_renderer.rs` — CORRECT, do not change |
| `screen_to_world` / `world_to_screen` | Operates in CSS pixel space only (no DPR factor) | `Viewport` struct in `renderer.rs` — CORRECT, consistent with shader |

**Convention document** (for Phase 13 interaction to rely on):
- World space: origin at (0,0), 1 unit = 1 CSS pixel at scale=1.0
- Screen space: origin at top-left of canvas CSS bounding box, 1 unit = 1 CSS pixel
- Pointer events: `MouseEvent.offset_x / offset_y` are CSS pixels — pass directly to `screen_to_world`
- Viewport offset at init: `(css_width/2, css_height/2)` — world origin maps to canvas center

### Force Simulation Contract

| Property | Required Behavior | Source |
|----------|-------------------|--------|
| Initial visibility | All nodes visible within canvas bounds on first render frame | Fix: reduce spread constant OR set fit-scale viewport |
| Spread formula fix | `let spread = (node_count as f64).sqrt() * 15.0` OR fit-to-viewport scale | `layout_state.rs` — current value 50.0 is too large |
| Fit-to-viewport alternative | `scale = min(1.0, css_width.min(css_height) * 0.45 / spread)` | Either fix is acceptable — executor chooses |
| Animation duration | Observable motion for first ~5 seconds (≥38 seconds total before convergence) | `ALPHA_DECAY=0.997`, alpha=1.0 → ~2296 frames at 60fps |
| Settlement | Nodes spread to distinct positions, no collapse to point or explosion off-screen | Barnes-Hut confirmed correct in 8 unit tests |

### Edge Rendering Contract

| Property | Value |
|----------|-------|
| Edges visible | All 720 edges drawn between their source and target nodes |
| Edge style | Thin line (1px world-space at scale=1.0) with arrowhead triangle at target |
| Edge color | `rgba(48, 54, 61, 0.6)` — `--color-border` at 60% opacity |
| Edge draw timing | Drawn in same `draw()` call as nodes — no separate pass required |

## Interaction States

The canvas renders three observable states. All are pre-existing — phase 12 must restore them to working.

| State | Trigger | Visual Outcome |
|-------|---------|----------------|
| Simulating | Graph page loads with data | Nodes visibly moving apart, alpha decaying from 1.0 toward 0.001 |
| Settled | Alpha < 0.001 (~38s at 60fps) | Nodes stationary in stable layout, edges visible |
| Loading | Server fetch in progress | `.graph-loading` spinner/text centered on canvas |

Note: Node drag, pan, and zoom interactions are deferred to Phase 13. Phase 12 must not introduce interaction regressions (canvas pointer-events must remain enabled so Phase 13 can attach handlers).
