Edit `resyn-app/style/main.css` to add pointer-events passthrough on overlay containers. Per D-01 and D-02, the `.graph-controls-overlay` and `.temporal-slider-row` intercept mouse events from reaching the canvas.

Specific changes (all additions, no deletions):

1. **`.graph-controls-overlay` rule (around line 1331):** Add `pointer-events: none;` to the existing rule block. This makes the overlay container transparent to mouse events so they pass through to the canvas beneath.

2. **`.graph-controls-group` rule (around line 1341):** Add `pointer-events: auto;` to re-enable pointer events on the button group container. Without this, the zoom +/-, fit, and simulation toggle buttons would stop working because `pointer-events: none` cascades to children.

3. **`.temporal-slider-row` rule (around line 1399):** Add `pointer-events: none;` to the existing rule block. This allows wheel and mouse events over the bottom slider bar to pass through to the canvas.

4. **`.temporal-range` rule (around line 1419):** Change `pointer-events: none;` to `pointer-events: auto;`. The range input element itself needs to accept pointer events so that the thumb pseudo-elements (which already have `pointer-events: all`) can function. Currently the input has `pointer-events: none` which works when the container accepts events, but after setting the container to `pointer-events: none`, the input itself must be `auto` to remain interactive.

Do NOT change:
- `.graph-tooltip` — already has `pointer-events: none` (correct)
- `.graph-control-btn` — inherits `pointer-events: auto` from `.graph-controls-group` (sufficient)
- `.temporal-range::-webkit-slider-thumb` — already has `pointer-events: all` (correct)
- `.temporal-range::-moz-range-thumb` — already has `pointer-events: all` (correct)

Do NOT touch any Rust files. The interaction logic, event handlers, coordinate transforms, and hit testing are all correct and unit-tested. Per D-03 through D-08, these should only be modified if live testing reveals issues after the CSS fix.
