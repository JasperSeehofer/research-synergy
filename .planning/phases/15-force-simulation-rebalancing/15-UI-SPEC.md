## Canvas Behavior Contract

This section defines the observable visual behavior that Phase 15 must produce. These are implementation acceptance criteria expressed as visual states.

### Initial Load State (frame 0)

- Nodes are positioned in concentric rings by BFS depth
- Depth-0 (seed paper): within 20px of canvas center
- Depth-1 nodes: on a ring of radius ~180–240px from center, spread evenly by angle
- Depth-2+ nodes: on progressively larger rings
- Orphan nodes (no BFS depth): on the outermost ring
- No nodes are overlapping at frame 0 (ring spacing ensures clearance)

### Spreading Animation (frames 1–~900)

- Nodes visibly move outward from their starting ring positions
- Motion is smooth and continuous — no jitter, no popping
- Hub nodes (high citation count, many edges) remain in their citation cluster rather than collapsing to center
- The spreading motion is satisfying to watch: nodes travel to distinct cluster regions over approximately 15–20 seconds at 60fps (~900–1200 ticks)
- Velocity clamping prevents scatter: no node travels faster than `IDEAL_DISTANCE / 2` per tick

### Converged State (frame ~900–1200)

- Nodes form visible loose clusters reflecting citation structure
- Papers with many shared citations are spatially closer than unrelated papers
- Clear negative space exists between citation subgraphs — clusters do not merge into a blob
- Node overlap is absent (collision separation force has resolved all overlaps)
- Simulation fully stops: no further node movement, CPU returns to idle

### Post-Drag State

- Dragged node moves to new position
- Simulation restarts locally (alpha reheated to ≥ 0.3)
- Nearby nodes rearrange smoothly
- Simulation returns to stopped state after local convergence
- The overall cluster structure does not re-randomize — only local rearrangement occurs

## Interaction Contract

### Simulation Pause/Resume Button (existing — no change)

- Element: `.graph-control-btn` with Unicode glyphs ⏸ (U+23F8) and ▶ (U+25B6)
- Location: `.graph-controls-overlay`, simulation control group
- Running state: shows ⏸ pause glyph
- Stopped/paused state: shows ▶ play glyph
- `aria-label`: "Pause force simulation" when running, "Resume force simulation" when stopped
- `aria-pressed`: reflects `simulation_running` boolean
- Visual: uses existing `.graph-control-btn` styles (32px min-width, 32px height, 12px label font, transparent background, `--color-surface-raised` on hover)
- No new CSS required

**Convergence indicator note:** LABEL-02 (convergence indicator) is scoped to Phase 17. Phase 15 does NOT add a convergence status display. The existing pause/resume button state is the only simulation feedback in this phase — when simulation stops, the button shows ▶ without any additional "Converged" label.

### Node Drag (existing — no change)

- Drag cursor: `grabbing` (existing `.graph-canvas:active` rule)
- Drag target: any non-pinned node
- On release: alpha reheat fires, `simulation_running` set to `true`, simulation restarts
- No new UI affordance for drag — behavior change is code-only
