| Node Type | Fill Color | Border Color | Notes |
|-----------|-----------|-------------|-------|

Note: Border color for regular nodes `#7cb8ff` is computed as the fill hue +30% brightness (brighter blue ring on blue fill). Exact value is Claude's discretion; `#7cb8ff` satisfies D-10 constraint. Seed border `#e8b84b` follows the same pattern.

## Renderer Visual Contract Summary

This section is the phase-specific addition to the standard UI-SPEC template. It captures the prescriptive visual rules the executor must implement.

### Canvas 2D Renderer (`canvas_renderer.rs`)

#### Edges
- Set `strokeStyle` to `rgba(139, 148, 158, {alpha})` where alpha is from the depth table above.
- Set `lineWidth` to `1.5`.
- Contradiction edges: `rgba(248, 81, 73, 1.0)`, `lineWidth = 2.0` (retain existing).
- Bridge edges: `rgba(210, 153, 34, 1.0)`, `lineWidth = 2.0` (retain existing).
- Arrowheads: same color and alpha as their parent edge.

#### Nodes
- Fill: solid `#4a9eff` for regular nodes, `#d29922` for seed node.
- Border arc: stroke with `#7cb8ff` (regular) or `#e8b84b` (seed). Line width = `1 / viewportScale` so it stays 1px screen-space at all zoom levels (D-13).
- No gradient, no shadow.

#### Seed Node Outer Ring
1. Draw fill circle at `radius`.
2. Draw border arc at `radius` (as above).
3. Skip gap (transparent — achieved by drawing nothing at `radius + 1px` to `radius + 2px`).
4. Draw outer ring: arc at `radius + 3px` (gap midpoint + ring thickness), `lineWidth = 3px`, color `#d29922`, alpha 1.0.

### WebGL2 Renderer (`webgl_renderer.rs`)

#### Node Fragment Shader (`NODE_FRAG`)
- Replace `smoothstep(0.9, 1.0, d)` with `fwidth(d)`-based computation for resolution-independent AA.
- Add border ring: thin annulus at `radius - border_width` using `fwidth` for crisp edge.
- Add `is_seed` uniform/attribute: when true, use amber fill `vec3(0.824, 0.600, 0.133)` instead of blue `vec3(0.290, 0.620, 1.0)`.
- Seed outer ring: additional annulus drawn at `radius + gap + ring_thickness/2` — or as a second draw call at the executor's discretion (D-08 pattern: separate pass is acceptable).

#### Edge Vertex Shader (`EDGE_VERT`) — Quad Geometry
- Input attributes: `a_position (vec2)`, `a_perp (vec2)`, `a_side (float)`, `a_color (vec3)`, `a_alpha (float)`, `a_half_width_px (float)`.
- Output varyings: `v_color (vec3)`, `v_alpha (float)`, `v_t (float)` — where `v_t` ranges -1.0 to +1.0 across the quad width.
- Perpendicular offset in NDC: `perp_ndc = a_perp * u_scale / u_resolution * 2.0 * a_half_width_px * a_side`.
- See RESEARCH.md Pattern 2 for full shader listing.

#### Edge Fragment Shader (`EDGE_FRAG`) — Quad Geometry
- Use `v_t` and `fwidth(v_t)` for soft anti-aliased edge borders.
- Alpha: `v_alpha * (1.0 - smoothstep(1.0 - fwidth(v_t), 1.0, abs(v_t)))`.
- Output: `fragColor = vec4(v_color, final_alpha)`.

#### Instance Data Format Changes
- Node instance: extend from 7 floats `[x, y, radius, alpha, r, g, b]` to 8 floats `[x, y, radius, alpha, r, g, b, is_seed]`.
- Edge vertex: extend from 6 floats `[x, y, r, g, b, alpha]` to 8 floats `[x, y, perp_x, perp_y, r, g, b, alpha]` (quad center + perpendicular direction).
