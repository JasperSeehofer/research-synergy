## Spacing Scale

Declared values inherited from existing design system (`--space-*` tokens in `main.css`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-xs`) | Icon gaps, button group padding, label pill internal gaps |
| sm | 8px (`--space-sm`) | Control group gap, label collision padding between bounding boxes |
| md | 16px (`--space-md`) | Overlay inset from viewport edge |
| lg | 24px (`--space-lg`) | Section padding (not used in this phase) |
| xl | 32px (`--space-xl`) | Layout gaps (not used in this phase) |
| 2xl | 48px (`--space-2xl`) | Major section breaks (not used in this phase) |
| 3xl | 64px (`--space-3xl`) | Page-level spacing (not used in this phase) |

Exceptions:
- Label pill height: 20px (11px font + 4.5px top + 4.5px bottom padding, rounded to nearest multiple of 4). Use literal `20.0f64` in Canvas 2D code.
- Label pill horizontal padding: 8px per side (equals `--space-sm`). Use literal `8.0f64`. Pill width formula: `text_w + 16.0` (8px each side).
- Label collision clearance: 8px between placed bounding boxes (literal `8.0f64` in `label_collision.rs`).
- Label-to-node gap: 8px below node bottom in screen space (`8.0f64`, equals `--space-sm`).
- Fit button: 32px × 32px (matches existing `.graph-control-btn` `min-width: 32px; height: 32px`).
- Status badge touch-/click-target: not interactive — display only.

Source: `main.css` `:root` tokens, `17-RESEARCH.md` Pattern 5 (greedy collision), `17-CONTEXT.md` D-10.

### Status Badge State Colors

| State | Text | CSS class modifier |
|-------|------|--------------------|
| Simulating... | `--color-text-muted` (#8b949e) | `.sim-status-badge.sim-running` |
| Paused | `--color-text-muted` (#8b949e) | `.sim-status-badge.sim-paused` |
| Settled | `--color-success` (#3fb950) | `.sim-status-badge.sim-settled` |

Settled uses `--color-success` (#3fb950) to signal natural completion positively. Simulating and Paused use `--color-text-muted` as a neutral, non-alarming indicator. ### Fit Button

- Element: `<button class="graph-control-btn">`
- Icon: `⤢` (U+2922, NORTH EAST AND SOUTH WEST ARROW). If U+2922 renders poorly in target browser, fallback to `⛶` (U+26F6).
- `aria-label`: `"Fit graph to viewport"`
- Position: appended to the simulation control group (`div.graph-controls-group`) after zoom − button
- Trigger: `on:click` increments `fit_count: RwSignal<u32>` (wrapping add)
- State: never `.active` — it has no toggle state

### Status Badge

- Element: `<span class="sim-status-badge sim-{state}">`
- Position: inside the node count indicator group (third `.graph-controls-group`), above the node count indicator
- Responsive text: three states mapped from two signals (`simulation_running`, `simulation_settled`)
- No interactive behavior — display only

### Canvas 2D Label Pills

- Drawn entirely in Rust/Canvas 2D — not HTML elements
- Screen-space rendering (after `ctx.set_transform(1,0,0,1,0,0)` resets world transform)
- Rounded rectangle via `ctx.round_rect_with_f64(x, y, w, h, 4.0)`
- Draw order: background fill → border stroke → text fill
- Hover override label: drawn after regular label pass, same pill style, no fade animation (instant show)

## Interaction Contract

### Auto-Fit Animation (D-01 through D-04)

| Property | Value |
|----------|-------|
| Trigger | First `check_alpha_convergence()` returning true, provided `user_has_interacted == false` |
| Duration | ~0.5s (per-frame lerp factor `t = 0.12` at 60fps produces exponential decay ease-out) |
| Easing | Exponential decay (per-frame lerp — no separate easing function needed) |
| Margin | 10% on each side — `margin_factor = 0.80` in `compute_fit_target()` |
| Stop condition | `|scale_delta| < 0.001` AND `|offset_delta| < 0.5px` for both axes |
| Label rendering during animation | Suppressed — no labels drawn while `fit_anim.active == true`. Resume on animation end. |
| Once-only guard | `fit_has_fired_once: bool` flag prevents re-trigger on subsequent convergences (e.g. drag reheat decay) |

### User Override Latch (D-05 through D-07)

| Event | Sets `user_has_interacted` |
|-------|---------------------------|
| Mouse pan (mousemove while `Panning` state) | Yes |
| Scroll wheel zoom | Yes |
| Zoom + button | Yes |
| Zoom − button | Yes |
| Node drag reheat | No (D-07) |
| Fit button | No (Fit is a manual re-trigger, not a user spatial adjustment) |

The `user_has_interacted` flag is permanent for the session. It is never reset except on full graph data reload.

### Fit Button (D-06, D-08)

- Triggers `compute_fit_target()` unconditionally (ignores `user_has_interacted`)
- Starts new `FitAnimState` with `active: true`
- Does not reset `user_has_interacted` — auto-fit remains latched after Fit button use
- RAF loop detects `fit_count` change and starts animation identically to auto-fit

### Label Collision Cache Invalidation (D-12)

| Condition | Action |
|-----------|--------|
| Viewport scale changed since last frame | Mark `label_cache_dirty = true` |
| Viewport offset changed since last frame | Mark `label_cache_dirty = true` |
| LOD visibility changed (zoom threshold) | Mark `label_cache_dirty = true` |
| Temporal slider moved | Mark `label_cache_dirty = true` |
| Graph load (initial) | Mark `label_cache_dirty = true` |
| Fit animation active (`fit_anim.active == true`) | Skip label rendering entirely — do not rebuild cache mid-animation |
| Fit animation just ended (`active` became `false`) | Mark `label_cache_dirty = true` — rebuild once |

`measureText` widths are cached at graph load time and are never invalidated by viewport changes. They are only invalidated if node data changes (e.g. graph reload).

### Hover Label Override (D-11)

- If `hovered_node == Some(i)` and `i` is NOT in `label_cache.visible_indices`: draw the hover label using same pill style, computed fresh from current viewport
- If `hovered_node == Some(i)` and `i` IS in `label_cache.visible_indices`: label is already visible — no additional draw needed
- Hover label is drawn after the regular label pass (higher paint order)
- Appearance: instant show, no fade animation (Claude's Discretion — fade adds complexity with marginal benefit for a utility graph tool)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Fit button aria-label | "Fit graph to viewport" |
| Fit button visible text | "⤢" (U+2922, no text label — icon only, consistent with zoom +/− buttons) |
| Status badge — running | "Simulating..." |
| Status badge — paused | "Paused" |
| Status badge — settled | "Settled" |
| Status badge aria-label | "Simulation status: {state}" (dynamic) |
| Empty state (no visible nodes) | Not applicable — fit skips gracefully with no user-visible message when zero nodes visible |
| Error state | Not applicable — no destructive actions or failure states in this phase |
| Destructive confirmation | None — this phase has no destructive actions |

Notes on copy choices (Claude's Discretion):
- "Simulating..." with ellipsis conveys ongoing process to the user.
- "Paused" is unambiguous — user explicitly stopped it.
- "Settled" is preferred over "Converged" (jargon) or "Done" (too generic). It communicates the force simulation found equilibrium.
- All three labels are single words (or one word + ellipsis) to fit within the compact controls overlay without wrapping.
