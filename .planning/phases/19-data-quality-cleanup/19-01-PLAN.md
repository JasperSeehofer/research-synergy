<objective>
Fix two data quality bugs in the InspireHEP pipeline: (1) published dates are never set on papers fetched from InspireHEP, breaking the temporal year-range slider, and (2) empty-string arXiv IDs from references without arxiv_eprint fields enter the BFS queue and create orphan nodes in the graph.

Purpose: After this plan, InspireHEP crawls produce papers with real published dates (enabling temporal filtering) and zero orphan nodes (every graph node has at least one edge). ORPH-01 (orphan diagnosis) is satisfied by the static analysis already documented in 19-RESEARCH.md.

Output: Modified inspirehep_api.rs (published date extraction + earliest_date in InspireMetadata + updated API field queries) and paper.rs (empty-ID filter). All existing and new tests pass.
</objective>

pub fn get_arxiv_references_ids(&self) -> Vec<String> {
    self.references
        .iter()
