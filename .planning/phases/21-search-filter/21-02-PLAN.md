**components/mod.rs** — Add `pub mod search_bar;` to the module declarations.

**components/search_bar.rs** — Create the GlobalSearchBar component. Key implementation details:

1. **Signals:**
   - `query: RwSignal<String>` — raw input value, updated on every keystroke
   - `debounced_query` — from `leptos_use::signal_debounced(query, 300.0)` — triggers server fetch
   - `results: Resource` — re-fetches when `debounced_query` changes, calls `search_papers(q, None).await`
   - `dropdown_open: RwSignal<bool>` — true when input focused AND (results exist OR loading)
   - `focused_idx: RwSignal<Option<usize>>` — keyboard arrow navigation index into results

2. **Input element:**
   - `<input>` with `NodeRef::<leptos::html::Input>` for Ctrl+K focus
   - `type="text"` placeholder `"Search papers..."`
   - `on:input` updates `query` signal
   - `on:focus` sets `dropdown_open = true`
   - `on:blur` sets `dropdown_open = false` after 200ms delay (allows click on dropdown)
   - `on:keydown` handles ArrowDown (increment focused_idx), ArrowUp (decrement), Enter (select focused result), Escape (close + clear)

3. **Ctrl+K global shortcut** (per D-04, RESEARCH.md Pattern 5):
   ```rust
   Effect::new(move |_| {
       let doc = web_sys::window().unwrap().document().unwrap();
       let input_ref_clone = input_ref.clone();
       let cb = Closure::<dyn FnMut(web_sys::KeyboardEvent)>::new(move |e: web_sys::KeyboardEvent| {
           if (e.ctrl_key() || e.meta_key()) && e.key() == "k" {
               e.prevent_default();
               if let Some(el) = input_ref_clone.get() {
                   let _ = el.focus();
               }
           }
       });
       doc.add_event_listener_with_callback("keydown", cb.as_ref().unchecked_ref()).unwrap();
       cb.forget(); // Intentional: lives for app lifetime
   });
   ```

4. **Dropdown rendering:**
   - Only rendered when `dropdown_open` is true AND `query` is non-empty
   - Show `"Searching..."` while resource is loading
   - Show `"No papers found"` + `"Try a different title, author, or keyword"` when results empty
   - For each result: two-line row (title on line 1, authors + year on line 2)
   - `on:mousedown` (not `on:click` — fires before blur) on result row triggers selection
   - Focused row (via arrow keys) gets `search-result-focused` class

5. **Result selection logic:**
   - On selection (click or Enter): read current page via `leptos_router::hooks::use_location`
   - Always: set `SelectedPaper` signal to open drawer with `paper_id`
   - If on `/graph` page: also set `SearchPanTrigger` signal with `paper_id`
   - Close dropdown, clear query
   - Per D-08: this is the ONLY point where SearchPanTrigger is set — never during typing

6. **Ctrl+K hint badge:**
   - `<kbd>` element showing "Ctrl+K" (or detect mac for "Cmd+K" if desired — Claude's discretion says this is fine either way)
   - Hidden when input is focused (use a derived signal from a focus state signal)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User keyboard input -> search query signal | Untrusted text enters reactive system |
| Search query -> server fn | Query string sent to server via Leptos server fn (serialized) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-04 | Tampering | GlobalSearchBar input | accept | Input is a search query — no mutation operations triggered. Server fn validates via parameterized DB query (T-21-01) |
| T-21-05 | DoS | Rapid keystroke flooding | mitigate | 300ms `signal_debounced` limits server fn calls to max ~3/second; empty-string guard on server prevents wasteful queries |
| T-21-06 | Spoofing | Ctrl+K event listener | accept | Global shortcut only focuses an input element — no privilege escalation possible |
</threat_model>
