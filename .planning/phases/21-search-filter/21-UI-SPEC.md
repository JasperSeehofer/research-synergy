## Component Inventory

New components introduced by this phase. Each maps to a decision in CONTEXT.md.

### 1. `GlobalSearchBar` — `resyn-app/src/components/search_bar.rs`

**Anatomy:**
```
[ 🔍 | input[placeholder="Search papers..."] | Ctrl+K ]
```

- Outer container: `display: flex; align-items: center; gap: var(--space-sm); background: var(--color-surface-raised); border: 1px solid var(--color-border); border-radius: var(--radius-md); padding: 0 var(--space-md); height: 36px;`
- Search icon: 16px, color `--color-text-muted`, flex-shrink 0
- Input: `flex: 1; background: transparent; border: none; outline: none; font-size: var(--font-size-body); color: var(--color-text);` — no border on the input itself, the container provides the visual border
- Focus state: container gains `outline: 2px solid var(--color-accent); outline-offset: 0;` (not the input — mirrors existing input:focus but on wrapper)
- Ctrl+K hint: `<kbd>` element, hidden when input is focused; `font-size: var(--font-size-label); color: var(--color-text-muted); background: var(--color-surface); border: 1px solid var(--color-border); border-radius: var(--radius-sm); padding: 0 8px;`
- Minimum width: 280px in full sidebar mode; 200px in rail mode

Source: D-01 (global bar in header); D-04 (Ctrl+K hint visible until focused)

**Placement in app shell:**
- Added to the sidebar header area or a new top bar between sidebar and content. The current `app-shell` is `display: flex` (sidebar + content-area). Insert a `.top-bar` row at the top of `content-area`: `display: flex; align-items: center; padding: var(--space-sm) var(--space-md); border-bottom: 1px solid var(--color-border); background: var(--color-surface); min-height: 48px;` — search bar sits inside this top bar.
- The top bar reduces `content-area` from full height by 48px.

Source: D-01; existing app-shell pattern

### 2. `SearchDropdown` — rendered within `GlobalSearchBar`

**Anatomy (result row):**
```
[ paper icon ] [ Title of the Paper (truncated 1 line)    ]
               [ Author One, Author Two · 2023            ]
```

- Dropdown panel: `position: absolute; top: calc(100% + var(--space-xs)); left: 0; right: 0; z-index: 200; background: var(--color-surface); border: 1px solid var(--color-border); border-radius: var(--radius-md); box-shadow: 0 8px 24px rgba(0,0,0,0.4); max-height: 320px; overflow-y: auto;`
- Each result row: `display: flex; flex-direction: column; gap: 2px; padding: var(--space-sm) var(--space-md); cursor: pointer; border-left: 3px solid transparent;`
- Hovered row: `background: var(--color-accent-muted); border-left-color: var(--color-accent);`
- Keyboard-focused row (arrow navigation): same as hover
- Title line: 14px, weight 400, color `--color-text`, `white-space: nowrap; overflow: hidden; text-overflow: ellipsis;`
- Authors + year line: 12px, weight 400, color `--color-text-muted`
- Empty state row: single row with "No papers found" at 14px, color `--color-text-muted`, padding `var(--space-md)`, no hover state
- Loading state: single row with "Searching…" at 14px, color `--color-text-muted`, padding `var(--space-md)`

Source: D-02 (top 8-10 matches with title, authors, year); CONTEXT.md — Claude's Discretion (dropdown styling)

### 3. `PapersFilterBar` — above papers table in `resyn-app/src/pages/papers.rs`

**Anatomy:**
```
[ 🔍 | input[placeholder="Filter papers..."] ]   [ 12 results ]
```

- Same visual style as `GlobalSearchBar` but smaller container: height 32px, minimum width unset (grows to fill available space above table)
- Result count badge: only shown when filter is non-empty; `font-size: var(--font-size-label); color: var(--color-text-muted); white-space: nowrap; margin-left: auto;`
- Placed in a flex row: `display: flex; align-items: center; gap: var(--space-md); margin-bottom: var(--space-lg);`
- Independent signal from GlobalSearchBar — does not share state (D-12)

Source: D-12 (separate inline filter bar); D-13 (server-side search, same debounce); CONTEXT.md — Claude's Discretion (result count badge)

### 4. Match Highlight in Table Cells

- When `filter_query` is non-empty, matching substring in title and author cells is wrapped with `<strong>` styled as: `font-weight: var(--font-weight-semibold); color: var(--color-accent);`
- Non-matching cell text remains `--color-text`
- Case-insensitive match, first occurrence only highlighted if multiple

Source: D-14

### 5. Graph Node Pulse Glow Ring — Canvas2D in `canvas_renderer.rs`

- Color: `--color-accent` (`#58a6ff`) — read as a constant in Rust, not via CSS
- Ring drawn as two additional `arc` calls after the node fill:
  - Outer ring 1: radius = `node_radius + 4 + 8 * sin(t * 3π)`, alpha = `(1.0 - t).max(0.0) * 0.8`, line-width 2.0
  - Outer ring 2: radius = `node_radius + 2`, alpha = `(1.0 - t).max(0.0) * 0.5`, line-width 1.0
- Duration: 120 frames at 60fps (~2s), 3 oscillations (D-06)
- After pulse: node retains no special styling — returns to normal render state
- Dimming of non-matched nodes during search: opacity factor 0.25 applied to `globalAlpha` before draw (mirrors existing `is_dimmed` pattern)

Source: D-06 (pulse glow ring, 2-3 pulses ~2s); D-07 (non-matching nodes dimmed); RESEARCH.md Pattern 6 and Pattern 7

## Interaction States

### Search Bar States

| State | Visual |
|-------|--------|
| Default (empty, unfocused) | Container border `--color-border`; placeholder text `--color-text-muted`; Ctrl+K badge visible |
| Focused (empty) | Container outline `2px solid --color-accent`; Ctrl+K badge hidden; no dropdown shown |
| Typing (debounce pending) | Container outline accent; dropdown shows "Searching…" if previous results exist, else empty |
| Results loaded | Dropdown shows up to 10 result rows |
| No results | Dropdown shows single "No papers found" row |
| Result hovered/focused | Row background `--color-accent-muted`; left border `--color-accent` |
| Result selected | Dropdown closes; drawer opens; graph pans (if on graph page) |
| Escape pressed | Dropdown closes; input retains focus; query cleared |

### Keyboard Navigation

| Key | Behavior |
|-----|----------|
| Ctrl+K / Cmd+K | Focus search bar from any page (D-04) |
| ArrowDown | Move focus to next result row |
| ArrowUp | Move focus to previous result row (wraps to input) |
| Enter | Select focused result row |
| Escape | Close dropdown, clear query, return focus to input |
| Tab | Close dropdown, move focus to next focusable element |

### Graph Pan Animation

| Phase | Visual |
|-------|--------|
| Triggered (result selected on graph page) | Lerp animation begins — smooth pan+zoom to center target node over ~400ms |
| During lerp | No pulse yet — node at normal render |
| Lerp complete | Pulse glow ring begins (2s, 3 oscillations) |
| Pulse complete | All non-matching nodes return to full opacity; highlighted node returns to normal |

Source: D-05 (lerp reuse); D-06 (pulse after pan); D-07 (dimming during search); D-08 (no pan while typing)

## Animation & Transition Contract

| Element | Property | Value | Notes |
|---------|----------|-------|-------|
| Dropdown open | `opacity` + `transform: translateY` | 150ms ease-out; from opacity 0 + translateY(-4px) to opacity 1 + translateY(0) | Consistent with existing drawer transition (200ms ease) |
| Dropdown close | `opacity` | 100ms ease-in; to opacity 0 | Faster close feels snappier |
| Result row hover | `background-color`, `border-left-color` | 100ms ease | Match existing nav-item hover (150ms ease) — slightly faster since rows are denser |
| Graph lerp pan | Position + scale | ~400ms, linear interpolation via existing `FitAnimState` | Reuse RESEARCH.md Pattern 4 / viewport_fit lerp |
| Graph pulse ring | Canvas2D arc alpha + radius | 2000ms total, 3 oscillations via `sin(t * 3π)` | 120 frames at 60fps; drawn inside render loop |
| Node dim (search active) | `globalAlpha` | Immediate (next frame) at 0.25; restore immediate on dropdown close | No transition — canvas redraws per frame |

Source: RESEARCH.md Pattern 5 (Ctrl+K); Pattern 6 (pulse ring); existing `--drawer-transition: 200ms ease` in main.css
