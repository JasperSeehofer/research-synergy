- Use `.spinner-sm` class (existing: 16px, 2px border, `--color-border` track, `--color-accent` leading edge)

## Component Inventory

New components required for this phase:

### 1. "Size by" Dropdown — `graph_controls.rs` addition

```
.graph-controls-group (existing wrapper)
  span.text-label "Size by"  ← or span.text-label + .spinner-sm when computing
  select.form-select (min-width: 120px)
    option value="uniform"     "Uniform"         (always enabled)
    option value="pagerank"    "PageRank"        (disabled + "Computing..." when unavailable)
    option value="betweenness" "Betweenness"     (disabled + "Computing..." when unavailable)
    option value="citations"   "Citations"       (always enabled)
```

State: `RwSignal<SizeMode>` where `SizeMode` enum: `Uniform | PageRank | Betweenness | Citations`

Disabled option label: append " (computing…)" when `metrics_status == Computing`, or " (unavailable)" when `metrics_status == NotAvailable`. Both at `--color-text-muted`.

### 2. "Most Influential Papers" Dashboard Card — `dashboard.rs` addition

Card structure follows existing `.dashboard-card` pattern with a ranked list body instead of a display number:

```
div.dashboard-card
  p.dashboard-card-title    "Most Influential Papers"
  div.influential-list      (new class — list of 5 entries)
    div.influential-entry   (per paper)
      span.influential-rank "1."   (12px semibold, --color-text-muted)
      div.influential-info
        p.influential-title  "{Paper Title}"   (14px regular, --color-text, truncate at 2 lines)
        p.influential-meta   "{year} · PR: 0.042"  (12px regular, --color-text-muted)
  a.dashboard-card-link     "View all →"
```

Skeleton state: 5 `.skeleton.skeleton-text` rows (height 14px) with `.skeleton.skeleton-text` for each entry.
Error state: `"—"` at `--color-text-muted` with `"Failed to load"` label (matches `ErrorCard` pattern).
Empty state (no papers crawled): reuse dashboard empty-state — does not show the card at all when `total_papers == 0`.
Computing state (metrics not yet available): show 5 skeleton rows with a `"Computing metrics…"` footer at 12px `--color-text-muted`, replacing "View all →".

### 3. Recompute Button (Claude's Discretion per CONTEXT.md)

Placement: inline in the "Size by" group, appended after the `<select>`. A secondary action button.

```
button.graph-control-btn  aria-label="Recompute graph metrics"
  "↺"   (Unicode ↺ U+21BA)
```

- Default state: visible, `--color-text-muted`, 32px height (matches `.graph-control-btn`)
- Computing state: `disabled`, show `.spinner-sm` replacing the "↺" glyph
- Tooltip (title attribute): `"Recompute centrality metrics"`

## Interaction Contracts

### Node Size Transition (CONTEXT.md D-02)

When user changes "Size by" dropdown value:
- Trigger lerp animation on node radius over 300ms
- Both Canvas 2D and WebGL2 renderers must honor: interpolate from current radius to target radius
- Easing: linear (matching existing `viewport_fit` lerp pattern)
- No transition when switching to/from `Uniform` with the same count — still lerp to avoid pop

### Tooltip Metric Score (CONTEXT.md D-03)

When a node is hovered and `SizeMode != Uniform`:
- Append metric line to existing tooltip overlay: `"PageRank: 0.0423"` or `"Betweenness: 0.1820"` or `"Citations: 14"`
- Format: `"{Metric Label}: {value}"` at 12px label weight 600 `--color-text`
- Position: below existing tooltip content, separated by 4px gap
- When `SizeMode == Uniform` or metric data unavailable for that node: omit the metric line entirely

### Disabled Dropdown Options (CONTEXT.md D-10)

- `PageRank` and `Betweenness` options render with `disabled` HTML attribute when `metrics_status != Ready`
- Label format while computing: `"PageRank (computing…)"`
- Label format when permanently unavailable: `"PageRank (unavailable)"`
- `Uniform` and `Citations` are never disabled

### Metrics Auto-Compute (CONTEXT.md D-07)

No UI feedback required from the user — auto-compute runs in the background. The only visible signal is:
- `.spinner-sm` next to the "Size by" label while computing
- Dropdown options transition from disabled to enabled when ready (no toast, no modal)
