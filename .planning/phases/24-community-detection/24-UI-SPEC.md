### Community Categorical Palette

Ten distinct community colors drawn from Tableau 10 (adapted for dark-background legibility). Applied as node fill colors in Canvas 2D and WebGL2 renderers. NOT CSS custom properties — passed as runtime color data to renderers.

| Index | Hex | Name |
|-------|-----|------|
| 0 | `#4e79a7` | Blue |
| 1 | `#f28e2b` | Orange |
| 2 | `#e15759` | Red |
| 3 | `#76b7b2` | Teal |
| 4 | `#59a14f` | Green |
| 5 | `#edc948` | Yellow |
| 6 | `#b07aa1` | Purple |
| 7 | `#ff9da7` | Pink |
| 8 | `#9c755f` | Brown |
| 9 | `#bab0ac` | Light Gray |
| "Other" | `#4a5568` | Neutral dark gray — for communities with < 3 papers |

Cycling rule: communities 10+ cycle modulo 10 through the above palette.
Community assignment to color index: ranked by community size descending (largest community = index 0).

**Source:** CONTEXT.md D-11 (Tableau 10 / D3 schemeCategory10 categorical palette) + D-04 (Other bucket neutral gray).

### Color Lerp Transition

Node fill colors lerp over **300ms** when `ColorMode` changes. Lerp operates in sRGB space per channel (R, G, B independently). Same animation tick as existing Phase 23 size lerp.

**Source:** CONTEXT.md D-12.

### BFS Depth Color Mode

When `ColorMode::BfsDepth` is active: use a warm→cool monotonic scale.

| Depth | Hex | Description |
|-------|-----|-------------|
| 0 (seed) | `#f28e2b` | Warm orange — seed paper, maximum salience |
| 1 | `#e15759` | Red |
| 2 | `#4e79a7` | Blue |
| 3 | `#76b7b2` | Teal |
| 4+ | `#8b949e` | Muted gray (clamp; depths beyond 4 share this) |
| no BFS depth | `#4a5568` | Orphan nodes — neutral dark gray |

**Source:** CONTEXT.md D-13. Exact values at discretion of researcher — this contract locks the warm→cool direction and the seed-is-most-salient principle.

### Topic Color Mode

When `ColorMode::Topic` is active: reuse the existing `palette: Vec<PaletteEntry>` data (r/g/b per keyword) already computed by the Topic Rings pipeline. The node fill color is derived from the paper's top keyword match in the palette. Papers with no matching palette entry use `#4a5568` (orphan gray).

**Source:** CONTEXT.md D-14 + existing `GraphState.palette` field in `layout_state.rs`.

## Component Inventory

### 1. ColorMode Dropdown (`graph_controls.rs`)

**Pattern:** Mirror of existing `SizeMode` `<select>` dropdown (lines 195–250 of `graph_controls.rs`).

```
Label: "Color by"  (12px / semibold / uppercase — matches "Size by" label)
Select element: class="graph-control-select"
Options:
  value="community"   → "Community"           (default, pre-selected — D-09)
  value="bfs_depth"   → "BFS Depth"
  value="topic"       → "Topic"
  value="community"   disabled when communities_ready=false → "Community (computing…)"
```

State: `RwSignal<ColorMode>` named `color_mode`. Placed directly below the `Size by` row in the controls group.

Computing state: When `communities_computing=true`, the "Community" option shows "Community (computing…)" and is `prop:disabled=true`. Spinner replaces the Recompute button label. This matches the Phase 23 `metrics_computing` pattern exactly.

### 2. Community Legend (`graph_controls.rs`)

**Pattern:** Mirror of existing `topic-legend-section` block (lines 252–297 of `graph_controls.rs`).

Rendered only when `color_mode.get() == ColorMode::Community`.

```
Section header: "COMMUNITY COLORS"
  class="sidebar-title"
  12px / semibold / uppercase / --color-text-muted
Entries container:
  class="topic-legend-entries"
  max-height: 160px, overflow-y: auto (existing pattern)
Each entry:
  class="legend-entry"
  Left: color swatch 12×12px, border-radius: 2px  (class="legend-swatch")
  Right: community keyword label (e.g. "quantum decoherence")
  Click → opens drawer on DrawerTab::Community for that community (D-16, legend-click mode)
  No active/selected state by default (legend chips are navigation, not filters)
"Other" entry (if present): always last, neutral swatch (#4a5568), label "Other"
```

Active state: `border-bottom: 2px solid var(--color-accent)`, color `--color-text`.

Displayed when a paper is selected AND `DrawerTab::Community` is active.

Layout (top to bottom within `.drawer-body`):

```

SECTION: "TOP PAPERS"  (12px / semibold / uppercase / muted — class="drawer-section-title")

  Each item:
    Paper title: 14px / regular, truncated at 2 lines with ellipsis

SECTION: "DOMINANT KEYWORDS"  (same section header class)
```

## Implementation Notes for Executor

1. **Leptos, not React.** All components are `.rs` files using `view! {}` macros. CSS classes reference `resyn-app/style/main.css` — no inline styles except for dynamic `background: rgb(r,g,b)` on swatches (established pattern from topic legend).

2. **Web app, not egui desktop.** CLAUDE.md describes the original egui desktop prototype; the actual UI is Leptos/WASM in `resyn-app/`. All Phase 21–23 UI work is Leptos.

3. **Color mode is data-driven, not CSS-driven.** Node fill colors are computed in Rust and passed to the Canvas 2D / WebGL2 renderers as `[u8; 3]` RGB values. The categorical palette hex values in this contract are the source of truth for those computed values. No CSS variables for node colors.

4. **Existing patterns to mirror precisely:**
   - `SizeMode` dropdown → `ColorMode` dropdown (same markup structure)
   - `topic-legend-section` → community legend section (same markup structure, different section header text)
   - `DrawerTab::Similar` addition (Phase 22) → `DrawerTab::Community` addition (this phase)
   - `metrics_computing` spinner pattern → `communities_computing` spinner pattern

5. **Color lerp implementation:** `current_color: [f32; 3]` and `target_color: [f32; 3]` fields per `NodeState`. Each frame: `current_color[i] += (target_color[i] - current_color[i]) * lerp_factor`. Lerp factor for 300ms at 60fps = `1 - (1 - 0.95)^(dt/300ms)` (exponential decay). Same tick as `current_radius` lerp.
