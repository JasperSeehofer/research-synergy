<objective>
Migrate OpenAlex authentication from deprecated mailto polite pool to API key bearer token.

Purpose: The OpenAlex polite pool (`?mailto=...`) was deprecated Feb 2026. Without an API key,
bulk-ingest hits a 100-credit daily cap and silently drops papers after ~20 pages. This replaces
`--mailto` with `--api-key` (backed by `OPENALEX_API_KEY` env var) and hard-fails if the key
is absent so failures are never silent.

Output: Updated `openalex_bulk.rs` and `bulk_ingest.rs` with bearer auth; all existing tests green.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → process env | `--api-key` accepts value via clap; env var `OPENALEX_API_KEY` is the preferred source so key never appears in `ps` output |
| process → OpenAlex API | API key transmitted in `Authorization: Bearer` header over HTTPS; never in URL query params |
| user-supplied filter → URL | `--filter` string is appended directly into the OpenAlex URL query string |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-01 | Information Disclosure | `fetch_page` URL builder | mitigate | Remove `?mailto=` from URL; pass key only as `Authorization: Bearer` header so key never appears in HTTP logs or shell history |
| T-27-02 | Information Disclosure | `tracing::info!` / `tracing::error!` calls | mitigate | Never pass `api_key` value as a tracing field; log only non-secret fields (filter, page count, paper count) |
| T-27-03 | Information Disclosure | `--api-key` CLI arg | mitigate | Use `env = "OPENALEX_API_KEY"` in clap so key is sourced from env, not argv visible in `ps aux` |
| T-27-04 | Tampering | `--filter` URL injection | accept | Filter is a developer-supplied constant string (`DEFAULT_FILTER`, `DEFAULT_FILTER_PHYSICS`); no user-facing input path; acceptable risk for a single-user CLI tool |
</threat_model>
