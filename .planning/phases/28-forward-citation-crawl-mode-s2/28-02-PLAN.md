    pub fn get_citing_arxiv_ids(&self) -> Vec<String> {
        self.citing_papers
            .iter()
            .filter_map(|r| r.get_arxiv_id().ok())
            .filter(|id| {
                if id.is_empty() {
                    tracing::warn!(
                        "Skipping empty arXiv ID in citing_papers for paper {}",
                        self.id
                    );
                    false
                } else {
                    true
                }
            })
            .collect()
    }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| serde_json deserializer | Untrusted JSON entering the `Paper` model from S2 / DB / file imports |
| Trait method dispatch | Code reuse boundary; default impls must be safe by construction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-07 | T (Tampering) — `citing_papers` injection via crafted JSON | `Paper` deserializer | accept | The field is `#[serde(default)]` — JSON sources can populate it, BUT plan 01's `fetch_citing_papers_inner` overwrites the field on every call, so any pre-populated value from a malicious JSON source is not persisted. DB serialization explicitly skips the field via `#[serde(skip_serializing)]`. A malicious JSON-import flow could populate it transiently, but no DB write path consumes it directly — only `upsert_inverse_citations_batch` (plan 03) writes edges, and it is called with explicit arguments, not from a `Paper` field |
| T-28-08 | I (Information Disclosure) — `citing_papers` leaking into JSON exports | export-louvain-graph, any future JSON exporter | mitigate | `#[serde(skip_serializing)]` ensures the field is never written by any `serde_json::to_*` call. Verified by Task 2 test `citing_papers_skipped_in_serialization` |
| T-28-09 | T (Tampering) — empty/oversized arxiv_id in `get_citing_arxiv_ids` | `Paper::get_citing_arxiv_ids` | mitigate | Empty-id filter mirrors `get_arxiv_references_ids` (existing pattern); oversized IDs are not validated here but downstream `paper_record_id` (queries.rs) and `validate_arxiv_id` (validation.rs) reject malformed IDs before any DB write |
| T-28-10 | E (Elevation of Privilege) — trait default impl bypassing override | `PaperSource::fetch_citing_papers` default impl | accept | Default impl is `Ok(())` no-op — strictly less privileged than any override. No EoP risk |
</threat_model>
