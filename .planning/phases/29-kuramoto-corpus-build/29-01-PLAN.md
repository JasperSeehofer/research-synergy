<objective>
Build the `data-kuramoto` corpus via bidirectional S2 crawl from 10 Feynman pair seeds, run analysis (NLP + Louvain community detection), export the pre-2015 graph for the notebook, and run the notebook as the phase success gate. Outcome: empirical pass/fail on the dynamical-LBD direction.
</objective>

## Step 1: Build release binary and write the 10-seed script

Build the release binary (~30-60s on warm cache):
```bash
cargo build --release --bin resyn
```

Write `scripts/crawl-feynman-pairs.sh`:
```bash
#!/usr/bin/env bash
# Phase 29: Kuramoto-LBD v03 corpus build
# Crawls 10 Feynman pair seeds with bidirectional S2 mode at depth 2.
set -euo pipefail

DB="${1:-surrealkv://./data-kuramoto}"
BINARY="${BINARY:-./target/release/resyn}"
MAX_FWD="${MAX_FWD:-500}"

seeds=(
  "cond-mat/0203227"   # pair01-A: Dorogovtsev Ising on networks
  "0710.3256"          # pair01-B: Castellano social dynamics
  "cond-mat/0010317"   # pair03-A: Pastor-Satorras SIR
  "cond-mat/0312131"   # pair03-B: Moreno rumour spreading
  "cond-mat/0007235"   # pair04-A: Newman random graphs
  "cond-mat/0205009"   # pair04-B: Newman epidemic disease
  "nlin/0202034"       # pair05-A: Drossel food webs
  "cond-mat/0002374"   # pair05-B: Bouchaud wealth
  "1005.1986"          # pair06-A: Nakao Turing
  "cond-mat/9801289"   # pair06-B: Marsili Zipf
)

echo "DB: $DB | Binary: $BINARY | max-fwd: $MAX_FWD"
echo "Seeds: ${#seeds[@]}"
echo ""

for ID in "${seeds[@]}"; do
  echo ""
  echo "=== Seeding $ID ==="
  "$BINARY" crawl \
    --paper-id "$ID" \
    --max-depth 2 \
    --source semantic_scholar \
    --bidirectional \
    --max-forward-citations "$MAX_FWD" \
    --db "$DB"
done

echo ""
echo "=== All 10 Feynman seeds complete ==="
```

`chmod +x scripts/crawl-feynman-pairs.sh`


## Step 4: Export pre-2015 graph

```bash
./target/release/resyn export-louvain-graph \
  --db surrealkv://./data-kuramoto \
  --output professional-vault/prototypes/data/research_synergy_pre2015.json \
  --published-before 2014-12-31 \
  --tfidf-top-n 50
```

**Pair-presence checkpoint** (Python one-liner):
```bash
python3 -c "
import json
d = json.load(open('professional-vault/prototypes/data/research_synergy_pre2015.json'))
ids = {n['id']: n['community_id'] for n in d['nodes']}
pairs = [('cond-mat/0203227','0710.3256'),
         ('cond-mat/0010317','cond-mat/0312131'),
         ('cond-mat/0007235','cond-mat/0205009'),
         ('nlin/0202034','cond-mat/0002374'),
         ('1005.1986','cond-mat/9801289')]
print(f'Nodes: {len(d[\"nodes\"])}  Edges: {len(d[\"edges\"])}')
n_eval = 0
for a,b in pairs:
    ca, cb = ids.get(a), ids.get(b)
    ok = ca is not None and cb is not None and ca != cb
    if ok: n_eval += 1
    print(f'  {a} (c={ca})  <->  {b} (c={cb})  [{\"OK\" if ok else \"FAIL\"}]')
print(f'n_eval = {n_eval} / 5  (gate: >= 3)')
"
```

If `n_eval < 3`, halt and write `29-VERIFICATION.md` with FAIL — do not run the notebook.
