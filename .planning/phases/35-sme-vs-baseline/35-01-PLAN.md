# Phase 35 — Plan 35-01 (EXP-RS-16): SME generator vs brute-force baseline

**Pre-registered.** Predictions P1–P4 + gate are LOCKED in `THREAD.md § Active experiment` and
must not be adjusted post-hoc. This plan fixes the *operational* knobs (K1–K7) before any method
runs; each is mirrored as an append-only row in `CONVENTIONS.md` (C-14…C-20). Scripts are committed
before they touch data (same discipline as Phases 29–34).

## Goal

Head-to-head, on the valid testbed, does a **Structure-Mapping (SME) generator over LLM-extracted
role-typed relational schemas** recover known cross-domain analogies at least as well as the
**brute-force LLM baseline (EXP-RS-10)** — with the *alignment table* as a co-primary, leakage-
resistant deliverable? Layer = semantic-conceptual (equations/mechanisms), NOT graph topology
(the killed dynamical/spectral family).

## Locked knobs (K1–K7 → conventions C-14…C-20)

### K1 — Corpus (MVP), `data/mvp_corpus.json`
- **Benchmark:** the 10 endpoint papers of the 5 evaluable Feynman pairs
  (`feynman_10pair_papers.json` evaluable_pairs = pair01,03,04,05,06 × side_a/side_b). All 10 are
  confirmed present in the testbed.
- **Distractors:** target **26**, sampled deterministically from
  `research_synergy_bridged_fine_sheaf.json`. Rule: exclude the 10 benchmark ids; iterate
  communities in ascending `community_id`; within a community, candidate ids sorted lexicographically;
  round *r* takes the *r*-th candidate from each community (round-robin across all 34 communities);
  skip any id whose abstract cannot be fetched (≥200 chars); stop at 26. Fully deterministic, no RNG,
  not tuned to the benchmark.
- **Abstracts:** arXiv API `id_list` (batched, 3 s rate-limit; old-format IDs like `cond-mat/…` and
  `nlin/…` work there). Fallback for any miss: OpenAlex `title.search`.
- Total ≈ **36 papers**, each with `{arxiv_id, title, abstract, community_id, is_benchmark, pair_id, side}`.

### K2 — Role vocabulary (closed)
Entity roles: `control-parameter, order-parameter, coupling, conserved-quantity, threshold,
state-variable, rate, interaction-network, external-field, noise-source, observable, other`.
Higher-order relation types: `CAUSES, UNDERGOES-TRANSITION-AT, CONSERVED-UNDER, COUPLES,
INCREASES-WITH, DECREASES-WITH, DEPENDS-ON`. Anything unmatched → `other`.

### K3 — Schema extraction (blind, abstract-only)
One Claude agent per paper sees ONLY `{title, abstract}` — no pair labels, no other papers, no
benchmark info (⇒ no scoring leakage). Structured output:
`{entities:[{id, role∈K2, placeholder, gloss}], relations:[{src, dst, type∈K2}]}`. Domain nouns
alpha-renamed to placeholders (X1, X2, …); `gloss` is a generic, domain-masked paraphrase. Output →
`data/mvp_schemas.json` (one schema per arxiv_id).

### K4 — Ablation arms (single extraction; matcher-side toggle)
- **roles-ON** (full SME): role-typed entities + typed relations; mapping must be role-consistent
  AND relation-type-consistent.
- **roles-OFF**: same schemas, entity **role labels stripped** (entities untyped); relation types
  kept; mapping relation-type-consistent only. Isolates the contribution of role typing. (Deriving
  roles-OFF by stripping labels — not a second LLM pass — removes extraction-variance confound and
  is a *stronger* control, since roles-OFF still inherits role-informed topology.)
- **lexical-null** (floor): abstract bag-of-words TF-IDF cosine, no schema. Doubles as the in-run
  "lexical overlap" control (the fixed TF-IDF floor is `BENCH_P10 > 0.15` per C-3).

### K5 — Systematicity score (`prototypes/sme_lite.py`)
For query schema G1 and candidate schema G2: build the association graph over entity-pairs
`(u1,u2)` that are role-compatible (roles-ON) or all-compatible (roles-OFF); connect two pairs when
the relation structure between them is type-consistent in both graphs. Max clique (Bron–Kerbosch,
networkx; schemas ≤ ~15 nodes) = maximum common role/relation-consistent mapping. **Systematicity =
number of relations preserved and mutually connected under that mapping** (isolated entity matches =
0 credit; deep interconnected relational overlap rewarded). Primary = connected preserved-relation
count. Ties → candidate arxiv_id lexicographic.

### K6 — Retrieval metric
Conditional retrieval. Primary: query = benchmark **side_a**, rank all other 35 papers by
systematicity desc → is **side_b** in top-k? Report `recall@{1,5,10}` and MRR over the 5 pairs.
Also report the reverse direction (side_b→side_a) and the both-direction average as robustness.
`BENCH_P10` (P2 floor) = recall@10 in the primary direction.

### K7 — Baseline (EXP-RS-10, job zero)
For each benchmark side_a, one Claude agent sees the side_a abstract + all 35 candidate
`{id,title,abstract}` and returns a ranked candidate list (structured output). Same recall@k / MRR
scoring as K6. This is the bar SME is judged against. (Known limitation: pretraining leakage inflates
the baseline; that is exactly why the alignment table is a co-primary deliverable.)

## Verdict gate (from THREAD — LOCKED, restated, not re-defined)
- **ADVANCE** iff P2 ∧ P3 ∧ P4: SME recall@10 ≥ baseline AND clears TF-IDF floor (BENCH_P10 > 0.15)
  AND roles-ON > roles-OFF AND alignment tables match `cross_bridges_ground_truth.json` bridge_names.
- **PIVOT** (not kill): SME ties baseline on recall@k but alignment tables high-quality → metric
  shifts to certified-mapping quality; CAS verifier becomes next build.
- **KILL SME**: fails TF-IDF floor OR roles-ON ≤ roles-OFF → fall back to slot-frames (#2) or
  mechanism-ontology (#4).

Secondary robustness (report, does not gate): rerun on `modern_lbd_pairs.json` (6 evaluable) to
blunt stat-phys overfit + pretraining leakage.

readouts — report all arms regardless of outcome.
