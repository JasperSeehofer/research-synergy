```rust
/// Result of k-means clustering on node positions.
pub struct ClusterResult {
    pub assignments: Vec<usize>,        // cluster index per node (parallel to input positions)
    pub centroids: Vec<(f64, f64)>,     // (x, y) per cluster
    pub dominant_keywords: Vec<String>, // top keyword per cluster (by summed TF-IDF)
}
```

**`auto_k(n: usize) -> usize`:**
- Formula: `((n as f64 / 2.0).sqrt().floor() as usize).clamp(3, 8)`
- If n == 0, return 3.

**`run_kmeans(positions: &[(f64, f64)], k: usize, max_iter: usize) -> Vec<usize>`:**
- Returns assignment vector (cluster index per node).
- If positions.len() <= k, assign each node its own cluster index (0, 1, 2, ...).
- K-means++ initialization:
  - First centroid: positions[0] (deterministic for reproducibility).
  - Each subsequent centroid: pick the point with maximum squared distance to its nearest existing centroid.
- Lloyd's iteration (max_iter iterations, default caller uses 20):
  - Assignment step: each point to nearest centroid (Euclidean distance).
  - Update step: each centroid = mean of assigned points. If a centroid has no assigned points, keep its previous position.
  - Early exit if no assignments changed.

**`dominant_keyword_for_cluster(node_indices: &[usize], all_keywords: &[Vec<(String, f32)>]) -> String`:**
- Sum TF-IDF scores by term across all nodes in the cluster.
- Return the term with the highest total score.
- If no keywords present, return empty string.

**`convex_hull(points: &[(f64, f64)]) -> Vec<(f64, f64)>`:**
- Jarvis march (gift-wrapping) algorithm.
- If fewer than 3 points, return the input as-is (cloned).
- Returns points in counter-clockwise order.
- Uses cross product to determine turn direction: `cross = (b.0 - a.0) * (c.1 - a.1) - (b.1 - a.1) * (c.0 - a.0)`. Positive = CCW turn (keep going), negative or zero = CW turn (candidate is more counter-clockwise).
- Start from the leftmost point (minimum x, break ties by minimum y).

**`score_to_opacity(score: f32) -> f64`:**
- Formula: `0.35 + (score as f64 * 0.65)` (per UI-SPEC opacity mapping)
- Clamp result to [0.35, 1.0].

**`compute_clusters(positions: &[(f64, f64)], all_keywords: &[Vec<(String, f32)>]) -> ClusterResult`:**
- Convenience function combining the above:
  - k = auto_k(positions.len())
  - assignments = run_kmeans(positions, k, 20)
  - For each cluster: compute centroid as mean of assigned node positions, compute dominant keyword.
  - Return ClusterResult.
