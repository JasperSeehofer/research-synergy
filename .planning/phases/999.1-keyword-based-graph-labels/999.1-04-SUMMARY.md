# Phase 999.1 Plan 04: Cluster Labels with Convex Hull and Extended Tooltip Summary

Cluster-level floating labels with dashed convex hull borders added (visible when scale < 0.6), with periodic cluster recompute during simulation and extended hover tooltip showing top-5 keywords with scores in keyword mode.

## What Was Built

### Task 1: draw_cluster_labels, cluster state, extended tooltip

**resyn-app/src/graph/label_collision.rs — draw_cluster_labels():**

- New `pub fn draw_cluster_labels()` function added after the `draw_not_analyzed_badge` function.
- For each cluster with 2+ member nodes: collects screen-space positions, computes convex hull via `crate::graph::kmeans::convex_hull()`, skips hulls with fewer than 3 points.
- Hull padding: each vertex moved 12px outward from the hull centroid using normalized direction vectors.
- Dashed hull border: `set_line_dash([4, 4])`, stroke style `rgba(88, 166, 255, 0.25)`, line width 1.5px. Subtle fill: `rgba(88, 166, 255, 0.04)`. Dash reset via empty `js_sys::Array` after stroke.
- Cluster label pill at `viewport.world_to_screen(centroid)`. Background `rgba(22, 27, 34, 0.9)`, border `rgba(88, 166, 255, 0.4)`, text `#e6edf3`. Uses arc_to rounded-rect path matching existing pill style.

**resyn-app/src/pages/graph.rs — RenderState cluster fields:**

- `cluster_result: Option<crate::graph::kmeans::ClusterResult>` — cached result, `None` when not in Keywords mode.
- `cluster_frame_counter: u32` — incremented each RAF frame in Keywords mode, used for 10-frame recompute cadence.
- Both initialized to `None` / `0` in `RenderState` construction.

**resyn-app/src/pages/graph.rs — RAF loop cluster recompute:**

Added between viewport change detection and label mode change detection:
- `just_converged` derived from `!sim_running && alpha <= ALPHA_MIN` for one-shot settle trigger.
- In Keywords mode: increments counter, recomputes when `cluster_result.is_none()` OR `(sim_running && counter % 10 == 0)` OR `just_converged`.
- Guard: only recomputes when `alpha < 0.15` to avoid degenerate clusters during initial chaotic layout.
- Collects positions and keywords from `lod_visible && temporal_visible` nodes only (consistent with what's rendered).
- Sets `cluster_result = None` when not in Keywords mode.

**resyn-app/src/pages/graph.rs — cluster rendering in label section:**

After the Keywords per-node pill loop, added:
```rust
if s.viewport.scale < crate::graph::lod::LOD_LEVEL_1
    && let Some(ref clusters) = s.cluster_result
{
    crate::graph::label_collision::draw_cluster_labels(ctx, clusters, &s.graph.nodes, &s.viewport);
}
```

**resyn-app/src/pages/graph.rs — extended node_tooltip():**

- `node_tooltip()` signature extended with `label_mode: &LabelMode` parameter.
- Appends `"\n────────────────────\nKeywords:"` and up to 5 `"\n  {term}  {score:.2}"` lines when `label_mode == Keywords && !node.top_keywords.is_empty()`.
- Single call site in mousemove handler updated to pass `&s.graph.label_mode`.
