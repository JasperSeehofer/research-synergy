### Topic Arc Palette (Canvas2D inline, 8-10 hues)

The topic arc palette encodes the top-N discriminative corpus keywords. Colors are perceptually distinct, dark-theme legible, and WCAG AA contrast compliant against `#0d1117` background.

Palette selection algorithm: evenly spaced hues in OKLCH color space at fixed lightness L=0.72, chroma C=0.18, hue step = 360 / palette_size. This produces perceptually uniform steps across the hue wheel while keeping all colors vibrant but not saturated to the point of harshness on a dark background.

Fixed palette (8 hues, evenly spaced OKLCH, approximated sRGB):

| Slot | Approximate Hue | sRGB Hex | Example keyword category |
|------|----------------|----------|--------------------------|
| 0 | 30° (amber/orange) | `#e8a34b` | Machine learning, Bayesian |
| 1 | 75° (yellow-green) | `#b8cc52` | Statistical mechanics |
| 2 | 120° (green) | `#56c76b` | Monte Carlo, simulation |
| 3 | 165° (teal) | `#40c9a4` | Quantum field theory |
| 4 | 210° (cyan-blue) | `#58b4e8` | Neural networks |
| 5 | 255° (blue-violet) | `#8884e0` | Graph theory |
| 6 | 300° (magenta) | `#cc6fd6` | String theory |
| 7 | 345° (rose) | `#e06080` | Cosmology |

Keywords beyond palette size 8 → neutral gray `#6e7681` (matches `--color-text-muted` shifted slightly lighter for arc visibility).

Palette cached in SurrealDB per D-03; keyword→slot assignment is deterministic (index into sorted discriminative keyword list).

### Node Arc State Colors

| Node/Arc State | Color | Notes |
|----------------|-------|-------|
| Topic arc segment | Palette slot hex (above) | Full opacity per-arc, variable arc length |
| Neutral remainder arc | `#3a424c` | Portion of ring where TF-IDF scores sum < 1.0; slightly lighter than `--color-bg`, darker than `--color-border` |
| Below-threshold node border | `#7cb8ff` (existing) | Plain 1px stroke — matches existing `canvas_renderer.rs` line 251 |
| Dashed ring (unanalyzed, above threshold) | `#6e7681` | Neutral; dashed `[4, 4]` pattern at 3px line width — signals data absence without color noise |
| Dimmed node topic arcs | Palette color at 0.3 opacity via `set_global_alpha(0.3)` | Applied when node is outside keyword filter selection; consistent with existing node dim logic |
| Active-filter dimmed arcs | `set_global_alpha(0.3)` on all non-matching nodes | Topic arcs follow node dim state from D-12 filter |

### Seed Node Ring Layering

Explicit draw order for seed nodes (from innermost to outermost):
1. WebGL2 filled circle (node fill `#d29922`)
2. Canvas2D topic arc segments at `radius` (3px line width, start top = `-π/2`, clockwise)
3. 2px transparent gap (no draw — gap is achieved by `ring_radius = radius + 2.0 + 1.5`)
4. Canvas2D planetary ring (`#d29922`, 3px line width) — existing code, drawn after topic arcs

Source: D-06; existing planetary ring code in `canvas_renderer.rs` lines 257-265.

## Interaction Contract

### Topic Rings Toggle (D-09)

| Property | Value |
|----------|-------|
| Element | `<button class="graph-control-btn">` — existing Leptos button component |
| Placement | `graph-controls-overlay`, new control group inserted between simulation controls and label mode dropdown |
| Label | `Topic Rings` |
| Default state | Enabled (`true`) — rings on by default; users can disable for a cleaner view |
| Active CSS class | `.graph-control-btn.active` — existing class, provides `--color-accent` background tint |
| ARIA | `aria-pressed={value}` — toggles between `"true"` / `"false"` |
| Signal | `RwSignal<bool>` named `show_topic_rings` — added to `GraphControls` component props |
| Effect on render | When `false`: skip `draw_topic_rings()`; render plain 1px border per existing code path |

### Topic Colors Legend (D-10, D-11, D-12)

| Property | Value |
|----------|-------|
| Placement | `graph-controls-overlay`, new section below "Topic Rings" toggle group |
| Section heading | `TOPIC COLORS` — 12px label, `--color-text-muted`, uppercase, matches `.sidebar-title` pattern |
| Visibility condition | Only rendered when `show_topic_rings == true` AND palette data is non-empty |
| Entry structure | `[colored swatch 12×12px] [keyword text 12px]` on one line, 8px horizontal gap |
| Entry height | 24px minimum (touch-accessible) |
| Scroll behavior | `overflow-y: auto; max-height: 160px` — scrollable when palette exceeds 6 visible entries |
| Active filter highlight | Selected legend entries get `background: var(--color-accent-muted)` (`#1f3a5f`) — distinguishes active filter state from hover |

### Legend Click-to-Filter (D-12)

| State | Visual | Behavior |
|-------|--------|----------|
| No filter active | All legend entries normal weight; all nodes full opacity | Default |
| One keyword selected | Clicked entry gets `--color-accent-muted` background; non-matching nodes dimmed to 0.3 opacity | Click entry to activate |
| Multiple keywords selected | All active entries highlighted; nodes matching ANY selected keyword stay full opacity (union) | Click additional entries to add |
| Clear filter | Click any active entry to deselect; if zero entries active, all nodes return to full opacity | Click again to toggle off |

Filter state is local to the page (not persisted). Signal: `RwSignal<HashSet<String>>` named `active_topic_filter`, initialized empty.

Dimming logic integrates with existing `is_dimmed()` closure in `canvas_renderer.rs` — a node is additionally dimmed if `active_topic_filter` is non-empty AND none of the node's `top_keywords` (top-3 for arcs) are in `active_topic_filter`.

### Topic Arc Rendering Threshold (D-13)

| Condition | Render |
|-----------|--------|
| `node.radius * viewport.scale >= 6.0` | Draw topic arcs (3px) |
| `node.radius * viewport.scale < 6.0` | Draw plain 1px border (`#7cb8ff` for regular, `#e8b84b` for seed) — existing code path |

Threshold constant: `MIN_SCREEN_RADIUS_FOR_RINGS: f64 = 6.0` — defined in `canvas_renderer.rs` (or a new `topic_rings.rs` module).

### Unanalyzed Node Dashed Ring (D-15)

| Property | Value |
|----------|-------|
| Condition | `node.top_keywords.is_empty() && node.radius * viewport.scale >= 6.0` |
| Color | `#6e7681` |
| Line width | 3px (same as topic arcs) |
| Dash pattern | `set_line_dash([4.0, 4.0])` — 4px on, 4px off |
| Draw call | `ctx.arc(x, y, radius, 0.0, TAU)` then `stroke()` — single full circle, dashed |
| After draw | `set_line_dash([])` — reset to solid for subsequent strokes |

### Arc Drawing Convention (Claude's Discretion)

| Property | Value |
|----------|-------|
| Start angle | `-π/2` (top of circle, 12 o'clock position) |
| Direction | Clockwise (positive angle increment) |
| Arc order | Keywords sorted by TF-IDF score descending; first keyword arc starts at top, each subsequent arc starts where previous ended |
| Angle per keyword | `2π × (score / total_score_sum)` where `total_score_sum = min(1.0, sum_of_top3_scores)` |
| Remainder arc | From end of last keyword arc back to `3π/2` (completing the circle) — rendered in neutral color `#3a424c` |

### Canvas2D Draw Order (D-08)

Topic arcs are drawn on the Canvas2D overlay, after WebGL2 node fill (or after Canvas2D fill if using Canvas2D renderer). Explicit per-frame draw order:

1. WebGL2 pass: draw all node fills (instanced quads)
2. Canvas2D overlay pass:
   a. Draw all edges (existing)
   b. Draw all node borders / topic arcs (new `draw_topic_rings()` or inline in border step)
   c. Draw labels / pills (existing)
   d. Draw seed planetary ring (existing — moved to after topic arcs)
   e. Draw selected/hovered outer ring (existing)

Note: Canvas2D renderer (non-WebGL path) draws fills and borders in the same pass — topic arcs replace the existing border `stroke()` call for above-threshold nodes.
