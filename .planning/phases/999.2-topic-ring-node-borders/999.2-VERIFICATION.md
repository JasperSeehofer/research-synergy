### Gaps Summary

Two gaps found, both from post-review design evolution in commit b071e92 ("post-review polish — remove legacy borders, scale-aware rings, clickable legend") that was applied after the plan was written but before the summary was finalized. The SUMMARY for Plan 02 does not document these deviations.

**Gap 1 — D-13 (below-threshold no-border):** The legacy 1px neutral border stroke was removed from `canvas_renderer.rs` to eliminate visual competition with topic arcs. However, D-13 requires that below-threshold nodes show a "plain 1px border instead." Currently below-threshold nodes show no border at all. Resolution: either restore the 1px fallback, or formally update D-13 to acknowledge no border is intentional.

**Gap 2 — D-14 (scale-aware vs. fixed 3px):** D-14 says ring width is fixed at 3px. The implementation uses proportional width (12% of screen radius, clamped 1.5–5px, +2.5px hover). This produces better visual output at various zoom levels but contradicts the spec. Resolution: either revert to fixed 3px, or update D-14 to reflect the intentional scale-aware behavior.

Both gaps are cosmetic spec-vs-implementation mismatches resulting from a deliberate design refinement. The core goal — colored arc segments encoding keyword fingerprints with legend and toggle — is fully implemented and functional at the code level.
